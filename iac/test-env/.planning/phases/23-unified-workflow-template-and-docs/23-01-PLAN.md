   - `deploy-step-function` job: same pattern as lambda but `if: needs.setup.outputs.resource_type == 'step_function'`, `concurrency.group: ferry-deploy-step-function`, uses `AmitLaviDev/ferry/action/deploy-stepfunctions@main`

   - NO workflow-level `concurrency:` key
   - All three deploy jobs use `strategy.matrix` with `fail-fast: false` (per CONTEXT.md Decision #1 -- matrix for ALL types)
   - Steps reference `${{ env.AWS_ROLE_ARN }}` and `${{ env.AWS_REGION }}` (not `${{ secrets.AWS_ROLE_ARN }}` directly in steps -- per CONTEXT.md Decision #2)
