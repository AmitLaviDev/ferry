### Critical Integration Gap: `function_name` Pipeline Break

**Problem:** `LambdaConfig.function_name` is validated from `ferry.yaml` but never reaches `deploy.py`.

**Data path (broken at step 2):**
1. `ferry.yaml` → `LambdaConfig` (has `function_name` with default-to-`name` validator) ✓
2. `_build_resource` in `trigger.py` → `LambdaResource` — **`function_name` DROPPED** ✗
3. `DispatchPayload` → `parse_payload.py` matrix — missing `function_name`
4. `deploy/action.yml` → `INPUT_FUNCTION_NAME` env var — never set
5. `deploy.py` line 165 → `os.environ["INPUT_FUNCTION_NAME"]` → **KeyError**

**Fix (3 files):**
- `utils/src/ferry_utils/models/dispatch.py` — add `function_name: str` to `LambdaResource`
- `backend/src/ferry_backend/dispatch/trigger.py` — pass `function_name=lam.function_name` in `_build_resource`
- `action/src/ferry_action/parse_payload.py` — include `"function_name": r.function_name` in lambda matrix
