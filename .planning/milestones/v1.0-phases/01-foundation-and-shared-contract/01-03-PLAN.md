<feature>
  <name>GitHub App JWT Generation</name>
  <files>backend/src/ferry_backend/auth/jwt.py, tests/test_backend/test_jwt.py</files>
  <behavior>
    generate_app_jwt(app_id: str, private_key: str) -> str

    Cases:
    - Valid app_id and PEM private key -> returns JWT string (not bytes — PyJWT 2.0+ returns str)
    - JWT has correct `iss` claim (app_id as string — per RESEARCH.md, use Client ID)
    - JWT has `iat` backdated 60 seconds from current time (clock drift tolerance)
    - JWT has `exp` set to current time + 540 seconds (9-minute buffer under 10-min max)
    - JWT uses RS256 algorithm
    - JWT is decodable with the corresponding public key
    - Invalid/malformed PEM key -> raises clear error

    Implementation details:
    - Use PyJWT: jwt.encode(payload, private_key, algorithm="RS256")
    - Claims: iss=app_id, iat=now-60, exp=now+540
    - Per RESEARCH.md: fresh JWT per webhook cycle, do NOT cache
    - Per RESEARCH.md anti-pattern: do not cache JWTs across invocations
  </behavior>
  <implementation>
    Single function using PyJWT. Generate claims dict with time-based fields, encode with RS256.
    See RESEARCH.md Pattern 3 for reference implementation.

    For testing: generate an RSA key pair in a pytest fixture using cryptography library (already installed as PyJWT[crypto] dependency). Decode the JWT with the public key to verify claims.
  </implementation>
</feature>

<feature>
  <name>Installation Token Exchange and GitHub Client</name>
  <files>backend/src/ferry_backend/auth/tokens.py, backend/src/ferry_backend/github/client.py, tests/test_backend/test_tokens.py</files>
  <behavior>
    get_installation_token(client: httpx.Client, jwt_token: str, installation_id: int) -> str

    Cases:
    - Valid JWT + installation ID -> returns installation access token string
    - Sends POST to /app/installations/{id}/access_tokens with correct headers
    - Requests scoped permissions: contents:read, checks:write, actions:write
    - Includes correct headers: Authorization: Bearer {jwt}, Accept: application/vnd.github+json, X-GitHub-Api-Version: 2022-11-28
    - GitHub returns 401 (bad JWT) -> raises GitHubAuthError with clear message
    - GitHub returns 404 (bad installation ID) -> raises GitHubAuthError
    - Network timeout -> raises with context

    GitHubClient class:
    - __init__(base_url: str = "https://api.github.com", timeout: float = 30.0)
    - Methods for authenticated requests: get(), post() that add standard headers
    - app_auth(jwt_token: str) -> sets Authorization: Bearer {jwt}
    - installation_auth(token: str) -> sets Authorization: token {token}
    - Thin wrapper — do NOT replicate httpx API. Just add auth headers and base URL.
    - ~150 lines max per MEMORY.md constraint

    Implementation details:
    - Use httpx.Client for HTTP calls
    - Per RESEARCH.md Pattern 3: POST /app/installations/{id}/access_tokens
    - Standard GitHub API headers on all requests
    - tenacity retry for 502/503 responses (GitHub infrastructure issues)
  </behavior>
  <implementation>
    GitHubClient wraps httpx.Client, adds base URL, standard headers, and auth.
    get_installation_token uses the client to exchange JWT for token.
    Tests use pytest-httpx to mock GitHub API responses.

    See RESEARCH.md Pattern 3 and pytest-httpx example for reference.
  </implementation>
</feature>
