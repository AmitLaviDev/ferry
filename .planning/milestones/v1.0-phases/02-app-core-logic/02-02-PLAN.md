  <action>
    **Behavior:**

    AffectedResource (frozen dataclass):
      - name: str (resource name from ferry.yaml)
      - resource_type: str ("lambda", "step_function", "api_gateway")
      - change_kind: str ("modified" or "new")
      - changed_files: list[str] (file paths that triggered this resource)

    get_changed_files(client, repo, base, head) -> list[str]:
      - Calls GET /repos/{repo}/compare/{base}...{head}
      - Extracts filename from each entry in response["files"]
      - Logs warning if len(files) == 300 (truncation signal from Compare API 300-file limit)
      - Returns list of file paths
      - When base is all zeros ("0" * 40), returns empty list (caller handles initial push via all-resources fallback)

    match_resources(config: FerryConfig, changed_files: list[str]) -> list[AffectedResource]:
      - For each resource in config.lambdas, config.step_functions, config.api_gateways:
        Check if any changed_file starts with resource.source_dir (with trailing slash normalization)
      - Returns AffectedResource with resource_type mapped from section name
      - change_kind is "modified" for all source_dir matches
      - Returns empty list if no matches

    merge_affected(source_affected, config_affected) -> list[AffectedResource]:
      - Merges two lists of AffectedResource by (name, resource_type) key
      - If same resource appears in both, keeps the one with more changed_files (or the config one if it's "new")
      - Deduplicates to prevent double-dispatch

    **Test cases (RED phase):**
      - test_get_changed_files_success: mock compare API -> returns file list
      - test_get_changed_files_empty: no files changed -> returns empty list
      - test_get_changed_files_truncation_warning: exactly 300 files -> logs warning (check structlog capture)
      - test_get_changed_files_initial_push: base is all zeros -> returns empty list (no API call)
      - test_match_resources_lambda_hit: file under lambda source_dir -> AffectedResource with type=lambda
      - test_match_resources_no_match: file outside all source_dirs -> empty list
      - test_match_resources_multiple_types: files matching lambda + step_function -> 2 AffectedResources
      - test_match_resources_trailing_slash: source_dir with/without trailing slash both work
      - test_match_resources_partial_prefix_no_match: source_dir "services/order" does NOT match "services/order-ext/main.py"
      - test_merge_affected_dedup: same resource in both lists -> single entry
      - test_merge_affected_no_overlap: different resources -> all kept

    **Implementation:**

    Create `backend/src/ferry_backend/detect/__init__.py` (empty).

    Create `detect/changes.py` with:

    AffectedResource as `@dataclass(frozen=True)` with fields: name, resource_type, change_kind, changed_files (tuple[str, ...] for immutability).

    get_changed_files: Check if base is all zeros first (return [] immediately). Otherwise call client.get with the compare URL. Extract filenames. Log warning via structlog if len == 300.

    match_resources: Iterate all three resource type sections from FerryConfig. For each resource, normalize source_dir to ensure trailing slash for prefix comparison (e.g., "services/order-processor" -> "services/order-processor/"). Use startswith for matching. Map section to resource_type string: lambdas->"lambda", step_functions->"step_function", api_gateways->"api_gateway".

    merge_affected: Build dict keyed by (name, resource_type). If conflict, prefer "new" change_kind, else keep the entry with more changed_files.

    Tests use pytest-httpx for mocking Compare API. For structlog warning test, use structlog.testing.capture_logs.

  <action>
    **Behavior:**

    detect_config_changes(old_config: FerryConfig | None, new_config: FerryConfig) -> list[AffectedResource]:
      - If old_config is None (ferry.yaml didn't exist before): all resources in new_config are "new"
      - Compares resource entries between old and new config:
        * Resource in new but not old -> "new"
        * Resource in old but not new -> ignored (removal, no dispatch)
        * Resource in both but fields differ -> "modified" (changed_files=("ferry.yaml",))
        * Resource in both and identical -> not affected
      - Comparison is by resource name within each type section
      - Returns list of AffectedResource entries

    **Test cases (RED phase):**
      - test_config_change_new_resource: resource in new but not old -> AffectedResource(change_kind="new")
      - test_config_change_removed_resource: resource in old but not new -> not in affected list
      - test_config_change_modified_resource: same name, different runtime -> AffectedResource(change_kind="modified", changed_files=("ferry.yaml",))
      - test_config_change_identical: same config -> empty affected list
      - test_config_change_no_old_config: old_config is None -> all resources are "new"
      - test_config_change_multiple_types: changes across lambdas and step_functions -> both detected

    **Implementation:**

    Add detect_config_changes to detect/changes.py:

    For each resource type section (lambdas, step_functions, api_gateways):
    - Build name->resource dicts for old and new
    - New names (in new_names - old_names): create AffectedResource with change_kind="new"
    - Removed names (in old_names - new_names): skip (no dispatch)
    - Common names: compare model_dump() dicts. If different, create AffectedResource with change_kind="modified", changed_files=("ferry.yaml",)

    The function receives pre-parsed, pre-validated FerryConfig objects (not raw YAML). The caller (handler.py in Plan 03) is responsible for fetching old/new configs.
