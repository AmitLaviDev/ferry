  <name>Feature 1: Config loader — fetch ferry.yaml from GitHub Contents API and parse YAML</name>
  <files>

  <action>
    **Behavior:**

    fetch_ferry_config(client, repo, sha) -> str:
      - Calls GET /repos/{repo}/contents/ferry.yaml?ref={sha}
      - Base64-decodes the content field from response JSON
      - Returns raw YAML string
      - Raises ConfigError("ferry.yaml not found in repository root") on 404
      - Raises ConfigError with status detail on other HTTP errors

    parse_config(raw_yaml: str) -> dict:
      - Calls yaml.safe_load on raw string
      - Returns parsed Python dict
      - Raises ConfigError("Malformed ferry.yaml: {detail}") on yaml.YAMLError

    **Test cases (RED phase):**
      - test_fetch_config_success: mock GET 200 with base64-encoded YAML content -> returns decoded string
      - test_fetch_config_not_found: mock GET 404 -> raises ConfigError with "not found" message
      - test_fetch_config_api_error: mock GET 500 -> raises ConfigError with status detail
      - test_parse_config_valid: valid YAML string -> returns dict with expected keys
      - test_parse_config_invalid: malformed YAML (e.g., "lambdas:\n  - [broken") -> raises ConfigError
      - test_parse_config_empty: empty string -> returns None (yaml.safe_load behavior); caller handles

    **Implementation:**

    Create `backend/src/ferry_backend/config/__init__.py` (empty).

    Create `config/loader.py` with two pure functions:
    - `fetch_ferry_config(client: GitHubClient, repo: str, sha: str) -> str`
      Uses `client.get(f"/repos/{repo}/contents/ferry.yaml", params={"ref": sha})`.
      On 404, raise ConfigError. On other errors, call resp.raise_for_status() wrapped in try/except httpx.HTTPStatusError -> ConfigError.
      Decode: `base64.b64decode(data["content"]).decode("utf-8")`.
    - `parse_config(raw_yaml: str) -> dict`
      Uses `yaml.safe_load(raw_yaml)`. Wraps yaml.YAMLError in ConfigError.

    Tests use pytest-httpx to mock GitHubClient responses. Import GitHubClient directly and use httpx_mock fixture.

    TDD: Write ALL tests first (RED commit), then implement (GREEN commit).

    **Behavior:**

    LambdaConfig (frozen, extra=forbid):
      - Required: name (str), source_dir (str), ecr_repo (str)
      - Optional: runtime (str, default "python3.10"), function_name (str|None, defaults to name via model_validator)
      - LambdaConfig(name="x", source_dir="s", ecr_repo="e") -> function_name="x", runtime="python3.10"
      - LambdaConfig(name="x", source_dir="s", ecr_repo="e", function_name="F") -> function_name="F"
      - LambdaConfig(name="x", source_dir="s") -> ValidationError (missing ecr_repo)
      - LambdaConfig(name="x", source_dir="s", ecr_repo="e", unknown="z") -> ValidationError (extra=forbid)

    StepFunctionConfig (frozen, extra=forbid):
      - Required: name (str), source_dir (str)
      - No optional fields

    ApiGatewayConfig (frozen, extra=forbid):
      - Required: name (str), source_dir (str)
      - No optional fields

    FerryConfig (frozen, extra=forbid):
      - version: int = 1
      - lambdas: list[LambdaConfig] = []
      - step_functions: list[StepFunctionConfig] = []
      - api_gateways: list[ApiGatewayConfig] = []
      - All sections optional (default to empty list)

    validate_config(raw: dict) -> FerryConfig:
      - Wraps FerryConfig.model_validate(raw)
      - On ValidationError, raises ConfigError with field-level error details

    IMPORTANT ferry.yaml structure: The YAML uses a list under each section, where each item has a `name` field. NOT a dict keyed by name. Example:
      lambdas:
        - name: order-processor
          source_dir: services/order-processor
          ecr_repo: ferry/order-processor

    **Test cases (RED phase):**
      - test_lambda_config_required_fields: all 3 required -> valid
      - test_lambda_config_defaults: function_name defaults to name, runtime defaults to python3.10
      - test_lambda_config_explicit_function_name: overrides default
      - test_lambda_config_missing_ecr_repo: raises ValidationError
      - test_lambda_config_extra_field: raises ValidationError (forbid)
      - test_step_function_config_valid: name + source_dir -> valid
      - test_api_gateway_config_valid: name + source_dir -> valid
      - test_ferry_config_full: all 3 sections with resources -> valid FerryConfig
      - test_ferry_config_empty: no sections -> valid FerryConfig with empty lists
      - test_ferry_config_partial: only lambdas section -> valid, others empty
      - test_validate_config_success: valid dict -> FerryConfig
      - test_validate_config_invalid: bad dict -> raises ConfigError (not raw ValidationError)
      - test_ferry_config_extra_section: unknown top-level key -> raises ValidationError

    **Implementation:**

    Create `config/schema.py` with:
    - `LambdaConfig(BaseModel)` with ConfigDict(frozen=True, extra="forbid"), required name/source_dir/ecr_repo, optional runtime="python3.10", optional function_name with model_validator(mode="after") defaulting to name (use object.__setattr__ for frozen model).
    - `StepFunctionConfig(BaseModel)` with ConfigDict(frozen=True, extra="forbid"), required name/source_dir.
    - `ApiGatewayConfig(BaseModel)` with ConfigDict(frozen=True, extra="forbid"), required name/source_dir.
    - `FerryConfig(BaseModel)` with ConfigDict(frozen=True, extra="forbid"), version=1, lambdas/step_functions/api_gateways as list types defaulting to [].
    - `validate_config(raw: dict) -> FerryConfig` wrapping model_validate, catching ValidationError, re-raising as ConfigError with str(exc) for field-level details.

    Follow Phase 1 pattern: all models frozen, ConfigDict on each.

    TDD: Write ALL tests first (RED commit), then implement (GREEN commit).
