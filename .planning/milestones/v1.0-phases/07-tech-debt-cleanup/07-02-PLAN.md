  <action>
    Create `docs/setup.md` covering shared concepts that apply to all resource types:

    1. **What is Ferry** -- 2-3 sentence overview: GitHub App + GitHub Action, detects changes from push, builds and deploys serverless resources.

    2. **Installation** -- Steps to install the Ferry GitHub App and add the Ferry Action to the repo.

    3. **ferry.yaml structure** -- Explain the top-level config file:
       - Lives in repo root as `ferry.yaml` (exact name, not `.ferry.yaml` or `ferry.yml`)
       - Version field (currently `1`)
       - Three top-level sections: `lambdas`, `step_functions`, `api_gateways`
       - Show a complete example ferry.yaml with all three resource types and inline comments
       - Explain that each section is optional (empty list if unused)

    4. **Workflow file naming convention** -- CRITICAL SECTION:
       - Ferry triggers workflows by name: `ferry-lambdas.yml`, `ferry-step_functions.yml`, `ferry-api_gateways.yml`
       - These names are derived from `RESOURCE_TYPE_WORKFLOW_MAP` in the codebase
       - The naming convention must match exactly or dispatches will fail silently (404)
       - Show the mapping: `lambda` -> `ferry-lambdas.yml`, `step_function` -> `ferry-step_functions.yml`, `api_gateway` -> `ferry-api_gateways.yml`

    5. **OIDC authentication** -- Explain the AWS OIDC model:
       - User creates an IAM role with GitHub OIDC trust policy
       - Role ARN is passed as an input to the Ferry actions
       - Ferry Action handles the STS AssumeRoleWithWebIdentity exchange
       - Brief note on required IAM permissions (ECR push, Lambda update, etc.)

    6. **How dispatch works** -- Brief explanation:
       - Ferry App detects changes per resource type
       - Fires one `workflow_dispatch` per resource type
       - The workflow receives a JSON payload with affected resources
       - The setup action parses the payload into a GHA matrix
       - Each matrix job builds/deploys one resource

    Cross-reference the per-resource-type docs at the bottom: "See [Lambda Workflows](lambdas.md), [Step Functions Workflows](step-functions.md), [API Gateway Workflows](api-gateways.md)".

    Use `python3.14` as the canonical runtime in all examples (the new default after Plan 01 changes).
