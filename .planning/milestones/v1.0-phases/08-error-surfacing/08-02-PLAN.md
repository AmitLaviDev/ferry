2. **gha.py** -- Add a `mask_account_id(account_id: str) -> str` function:
```python
def mask_account_id(account_id: str) -> str:
    """Mask an AWS account ID to show only the last 4 digits.

    Args:
        account_id: The full AWS account ID string.

    Returns:
        Masked string like '****1234'.
    """
    if len(account_id) > 4:
        return f"****{account_id[-4:]}"
    return account_id
```
This is used by build/deploy modules when constructing error messages.

3. **report.py** -- Create NEW module `action/src/ferry_action/report.py` with:

   a. `report_check_run(resource_name, phase, conclusion, summary, trigger_sha)`:
      - `phase` is "build" or "deploy" (string)
      - `conclusion` is "success" or "failure"
      - `summary` is the terse message (error message on failure, brief success note on success)
      - Read `GITHUB_TOKEN` from env (via `os.environ.get("GITHUB_TOKEN", "")`) and `GITHUB_REPOSITORY` from env
      - If either is missing, log a warning and return silently (graceful skip outside GHA context)
      - Check Run name: `f"Ferry: {resource_name} {phase}"` (per CONTEXT.md -- resource-specific titles)
      - Title: `f"{phase.capitalize()} {'succeeded' if conclusion == 'success' else 'failed'}"`
      - POST to `https://api.github.com/repos/{repo}/check-runs` with `httpx.post()`
      - Headers: `Authorization: token {token}`, `Accept: application/vnd.github+json`, `X-GitHub-Api-Version: 2022-11-28`
      - JSON body: `name`, `head_sha` (trigger_sha), `status: completed`, `conclusion`, `output: {title, summary}`
      - Wrap in try/except to catch httpx errors -- log warning but do NOT fail the build/deploy if Check Run creation fails (non-critical)
      - Timeout: 30.0 seconds

   b. `format_error_detail(exc, hint)`:
      - Takes an exception and a human-readable hint string
      - Returns the hint by default
      - If `FERRY_DEBUG` env var is set to `1`, `true`, or `yes` (case-insensitive), append `\n\nFull traceback:\n{traceback.format_exc()}`
      - This is used by build/deploy modules when formatting error messages
