## Overview

v1.5 replaced per-type workflow_dispatch (one dispatch per resource type, producing N workflow runs with skipped-job clutter) with a single batched dispatch per push. One push produces one workflow run with only the relevant deploy jobs active.

## Key Accomplishments

1. **BatchedDispatchPayload v2 model** — typed per-type resource lists with Literal[2] version discriminator
2. **Single dispatch per push** — replaced N API calls (one per type) with 1 batched call
3. **Automatic >65KB fallback** — oversized payloads fall back to per-type v1 dispatch
4. **Per-type boolean flags + matrices** — clean GHA job routing without empty-matrix crashes
5. **v1/v2 backward compatibility** — action parses both payload formats during rollout
6. **E2E validated** — single-type (1 active, 2 skipped) and multi-type (3 active, 0 skipped) proven in production
