**Core Value:** When a developer pushes code, every affected serverless resource is automatically detected, built, and deployed -- with full visibility on the PR before merge.

## v1.5 Requirements

### Dispatch

- [x] **DISP-01**: Backend sends a single workflow_dispatch per push containing all affected resource types in one payload — Phase 26
- [x] **DISP-02**: Batched payload uses schema version field (v=2) to distinguish from v1 per-type payloads — Phase 25
- [x] **DISP-03**: Backend falls back to per-type dispatch if combined payload exceeds 65,535 character limit — Phase 26

### Action

- [x] **ACT-01**: Setup action outputs per-type boolean flags (has_lambdas, has_step_functions, has_api_gateways) — Phase 27
- [x] **ACT-02**: Setup action outputs per-type matrix JSON (lambda_matrix, sf_matrix, ag_matrix) — Phase 27
- [x] **ACT-03**: Setup action outputs resource_types string (comma-separated list of affected types) for run-name — Phase 27
- [x] **ACT-04**: Setup action parses both v1 (per-type) and v2 (batched) payloads for backward compatibility during rollout — Phase 27

### Template

- [x] **TMPL-01**: Deploy jobs gate on boolean flags (if: has_lambdas == 'true') instead of resource_type string comparison — Phase 27
- [x] **TMPL-02**: Each deploy job references its own per-type matrix output (fromJson(needs.setup.outputs.lambda_matrix)) — Phase 27
- [x] **TMPL-03**: Workflow run-name dynamically displays all affected resource types — Phase 27

### Validation

- [x] **VAL-01**: Multi-type push (touching all 3 types) produces 1 workflow run with all 3 deploy jobs active — Phase 28 (run 22975664942)
- [x] **VAL-02**: Single-type push produces 1 workflow run with 1 active deploy job and 2 skipped — Phase 28 (run 22975593340)
