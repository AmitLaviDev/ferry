Create `docs/setup-runbook.md` with the following sections. Per user decision: scope is Phase 14 manual steps only (not Phases 11-13 apply order). Audience is us/future contributors who have AWS access and Terraform familiarity.

**Section 1: Prerequisites**
- AWS credentials configured (`aws sts get-caller-identity` works)
- Phases 11-13 Terraform applied (Lambda Function URL exists)
- Function URL from `terraform output` in `iac/aws/staging/us-east-1/ferry_backend/`
- `ferry-gha-self-deploy` IAM role ARN from `terraform output` in `iac/aws/staging/shared/`

**Section 2: Register GitHub App**
Step-by-step at github.com/settings/apps:
1. Go to Settings > Developer settings > GitHub Apps > New GitHub App
2. App name: "Ferry" (or "Ferry-Staging" etc.)
3. Homepage URL: the repo URL
4. Webhook URL: paste the Function URL from Phase 13 output
5. Webhook secret: generate with `openssl rand -hex 20` (save this value for Step 3)
6. Repository permissions:
   - Contents: Read
   - Pull requests: Read & Write
   - Checks: Read & Write
   - Actions: Write
   - Metadata: Read (always required, auto-selected)
7. Subscribe to events: Push
8. Where can this GitHub App be installed? "Only on this account" (for now)
9. Click "Create GitHub App"
10. Note the **App ID** from the app settings page
11. Generate a private key: scroll to "Private keys" > "Generate a private key" > save the `.pem` file
12. Install the App on the ferry repo: Settings > Install App > select the ferry repo

Note the **Installation ID** from the URL after installation (https://github.com/settings/installations/{ID}).

**Section 3: Populate Secrets Manager**
Three commands using the values from Step 2:
```bash
aws secretsmanager put-secret-value \
  --secret-id ferry/github-app/app-id \
  --secret-string "<APP_ID>"

aws secretsmanager put-secret-value \
  --secret-id ferry/github-app/private-key \
  --secret-string "$(cat /path/to/ferry-app.private-key.pem)"

aws secretsmanager put-secret-value \
  --secret-id ferry/github-app/webhook-secret \
  --secret-string "<WEBHOOK_SECRET>"
```

**Section 4: Configure GitHub Repo Secrets**
Set the `AWS_DEPLOY_ROLE_ARN` repo secret:
1. Go to ferry repo > Settings > Secrets and variables > Actions
2. New repository secret: `AWS_DEPLOY_ROLE_ARN`
3. Value: the ARN of the `ferry-gha-self-deploy` role from `terraform output gha_self_deploy_role_arn` in `iac/aws/staging/shared/`

**Section 5: Update Installation ID**
Update the Lambda env var with the actual installation ID. Two options:
- Option A (quick): `aws lambda update-function-configuration --function-name ferry-backend --environment "Variables={...all existing vars...,FERRY_INSTALLATION_ID=<ID>}" --no-cli-pager`
- Option B (permanent): Update `installation_id` default in `iac/aws/staging/us-east-1/ferry_backend/variables.tf` and run `terraform apply`

**Section 6: Trigger First Deploy**
Push a commit to main (or merge a PR). The self-deploy workflow should:
1. Run tests
2. Build and push Docker image to ECR
3. Update Lambda function code

Monitor at: `https://github.com/<owner>/ferry/actions/workflows/self-deploy.yml`

**Section 7: Verify End-to-End**
Per user decision, include these verification steps:
1. Curl the Function URL: `curl -s https://<function-url>/ | jq .`
   - Expected: JSON response (even error is fine -- proves Lambda is running the real code)
2. Send test webhook: Go to GitHub App settings > Advanced > Recent Deliveries > click "Redeliver"
   - Expected: 200 or 401 (signature mismatch on redeliver is expected) -- any response proves Lambda is handling webhooks
3. Check CloudWatch logs: `aws logs tail /aws/lambda/ferry-backend --since 5m --follow`
   - Expected: Structured JSON log entries showing webhook processing
