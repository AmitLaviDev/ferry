Create `.github/workflows/` directory if it doesn't exist, then create `self-deploy.yml`.

**Workflow structure:**

```yaml
name: Self-Deploy Backend
on:
  push:
    branches: [main]
```

Per user decision: triggers on **every push to main**, no path filtering.

**Permissions** (workflow-level):
```yaml
permissions:
  id-token: write   # Required for OIDC token request
  contents: read     # Required for actions/checkout
```

**Job 1: `test`** (per user decision: run pytest before build)
- `runs-on: ubuntu-latest`
- Steps:
  1. `actions/checkout@v4`
  2. `actions/setup-python@v5` with `python-version: "3.14"`
  3. Install uv: `pip install uv`
  4. Install deps: `uv sync --frozen`
  5. Run tests: `uv run pytest`

**Job 2: `deploy`**
- `needs: test` (per user decision: build+deploy depends on tests passing)
- `runs-on: ubuntu-latest`
- `environment: staging` (optional, for GitHub deployment protection rules later)
- Steps:
  1. `actions/checkout@v4`
  2. `aws-actions/configure-aws-credentials@v6` with:
     - `role-to-assume: ${{ secrets.AWS_DEPLOY_ROLE_ARN }}`  (the `ferry-gha-self-deploy` role ARN, stored as repo secret)
     - `aws-region: us-east-1`
  3. `aws-actions/amazon-ecr-login@v2` with `id: ecr-login`
  4. `docker/setup-buildx-action@v3`
  5. `docker/build-push-action@v6` with:
     - `context: .`
     - `push: true`
     - `tags: ${{ steps.ecr-login.outputs.registry }}/lambda-ferry-backend:${{ github.sha }}`
     - `cache-from: type=gha`
     - `cache-to: type=gha,mode=max`
  6. Update Lambda (run step):
     ```
     aws lambda update-function-code \
       --function-name ferry-backend \
       --image-uri "$IMAGE_URI" \
       --no-cli-pager
     aws lambda wait function-updated-v2 \
       --function-name ferry-backend
     ```
     With env: `IMAGE_URI: ${{ steps.ecr-login.outputs.registry }}/lambda-ferry-backend:${{ github.sha }}`

Key details:
- ECR repo name is `lambda-ferry-backend` (from `iac/global/cloud/aws/ecr/variables.tf` default)
- Lambda function name is `ferry-backend` (from `iac/aws/staging/us-east-1/ferry_backend/variables.tf` default)
- Use `function-updated-v2` waiter (1s poll, 300 retries) not the old `function-updated` (5s poll, 60 retries)
- `--no-cli-pager` prevents interactive pager from blocking CI
- Image tagged with `${{ github.sha }}` for traceability
- Per user decision: fail loudly, no rollback -- if deploy fails, Lambda keeps running previous image

The `AWS_DEPLOY_ROLE_ARN` repo secret must be set manually to the ARN of the `ferry-gha-self-deploy` IAM role. This will be documented in the setup runbook (Plan 03).
