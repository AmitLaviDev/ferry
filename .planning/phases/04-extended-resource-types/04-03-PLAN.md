GREEN: Create `action/src/ferry_action/deploy_apigw.py`:

```python
"""API Gateway deployment with OpenAPI spec upload and content-hash skip.

Deploys an OpenAPI spec to an API Gateway REST API:
1. Reads spec file (JSON or YAML) from source_dir.
2. Performs envsubst (${ACCOUNT_ID}, ${AWS_REGION}).
3. Parses to dict, strips problematic fields (host, schemes, basePath, servers).
4. Serializes to canonical JSON for hashing and upload.
5. Checks content hash against ferry:content-hash tag (skip if unchanged).
6. Calls put_rest_api (mode=overwrite) + create_deployment.
7. Updates the content hash tag.

Invoked as: python -m ferry_action.deploy_apigw
"""
```

Functions:
- `strip_openapi_fields(spec: dict) -> dict`: Remove `host`, `schemes`, `basePath` (Swagger 2.0) and `servers` (OpenAPI 3.x). Shallow copy, pop fields. Per Claude's Discretion, strip `servers` too since API Gateway manages endpoints via stages.
- `should_skip_deploy(apigw_client, rest_api_id: str, region: str, new_hash: str) -> bool`: Construct ARN `arn:aws:apigateway:{region}::/restapis/{rest_api_id}`. Call `get_tags(resourceArn=arn)`. Extract hash from `tags` dict (flat `{"key": "value"}` format for APIGW). Compare. Catch `ClientError` and return False (deploy if tags unavailable).
- `deploy_api_gateway(apigw_client, rest_api_id: str, stage_name: str, spec_body: bytes, canonical_json: str, region: str, deployment_tag: str) -> dict`: Call `put_rest_api(restApiId=rest_api_id, mode="overwrite", failOnWarnings=False, body=spec_body)`. Then `create_deployment(restApiId=rest_api_id, stageName=stage_name, description=f"Deployed by Ferry: {deployment_tag}")`. Then `tag_resource` with `ferry:content-hash`. Return `{"deployment_id": deploy_resp["id"], "rest_api_id": rest_api_id, "stage": stage_name, "skipped": False}`.
- `main()`:
  1. Read env vars: INPUT_RESOURCE_NAME, INPUT_REST_API_ID, INPUT_STAGE_NAME, INPUT_SPEC_FILE, INPUT_SOURCE_DIR, INPUT_DEPLOYMENT_TAG, INPUT_TRIGGER_SHA.
  2. Create STS client, get account_id.
  3. Get region from AWS_DEFAULT_REGION or AWS_REGION.
  4. Read spec file from `Path(source_dir) / spec_file`.
  5. Apply envsubst to raw content.
  6. Parse: if extension is `.yaml` or `.yml`, use `yaml.safe_load`; otherwise `json.loads`.
  7. Strip problematic fields.
  8. Serialize to canonical JSON: `json.dumps(spec_dict, sort_keys=True, separators=(",", ":"))`.
  9. Compute content hash of canonical JSON.
  10. Check skip via should_skip_deploy.
  11. If skip: GHA outputs (skipped=true), summary, return.
  12. Deploy: call deploy_api_gateway with `canonical.encode("utf-8")` as body.
  13. GHA outputs (skipped=false, deployment-id=...).
  14. GHA summary table: resource name, REST API ID, stage, deployment ID, status, deployment tag.
  15. Error handling: catch ClientError with hints for `NotFoundException` ("REST API '{rest_api_id}' not found"), `BadRequestException` ("Invalid OpenAPI spec -- check spec syntax"), `AccessDeniedException` ("IAM role lacks apigateway permissions").

All within `gha.begin_group` / `gha.end_group`.

**Important:** The `body` parameter of `put_rest_api` MUST be bytes (`.encode("utf-8")`), not a string. This is a known pitfall from research.

**Important:** For content hashing, always hash the canonical JSON (after parse + strip + re-serialize with sort_keys). This ensures deterministic hashing regardless of original key ordering or format (JSON vs YAML).

Run tests -- they MUST pass.

Create `action/deploy-apigw/action.yml` following the exact same pattern as `action/deploy/action.yml` and `action/deploy-stepfunctions/action.yml`:

```yaml
name: "Ferry Deploy API Gateway"
description: "Deploy API Gateway from OpenAPI spec"

inputs:
  resource-name:
    description: "Name of the API Gateway resource"
    required: true
  rest-api-id:
    description: "REST API ID in AWS"
    required: true
  stage-name:
    description: "Stage name to deploy to"
    required: true
  spec-file:
    description: "OpenAPI spec file name relative to source-dir"
    required: true
  source-dir:
    description: "Source directory containing the spec file"
    required: true
  trigger-sha:
    description: "Git SHA that triggered the deployment"
    required: true
  deployment-tag:
    description: "Deployment tag (e.g. pr-42)"
    required: true
  aws-role-arn:
    description: "IAM role ARN for OIDC authentication"
    required: true
  aws-region:
    description: "AWS region"
    required: false
    default: "us-east-1"

outputs:
  skipped:
    description: "Whether deploy was skipped (spec unchanged)"
    value: ${{ steps.deploy.outputs.skipped }}
  deployment-id:
    description: "API Gateway deployment ID"
    value: ${{ steps.deploy.outputs.deployment-id }}

runs:
  using: composite
  steps:
    - name: Configure AWS credentials via OIDC
      uses: aws-actions/configure-aws-credentials@v4
      with:
        role-to-assume: ${{ inputs.aws-role-arn }}
        aws-region: ${{ inputs.aws-region }}

    - name: Install uv
      uses: astral-sh/setup-uv@v4

    - name: Install ferry-action package
      shell: bash
      run: uv pip install --system "${{ github.action_path }}/.."

    - name: Deploy API Gateway
      id: deploy
      shell: bash
      env:
        INPUT_RESOURCE_NAME: ${{ inputs.resource-name }}
        INPUT_REST_API_ID: ${{ inputs.rest-api-id }}
        INPUT_STAGE_NAME: ${{ inputs.stage-name }}
        INPUT_SPEC_FILE: ${{ inputs.spec-file }}
        INPUT_SOURCE_DIR: ${{ inputs.source-dir }}
        INPUT_TRIGGER_SHA: ${{ inputs.trigger-sha }}
        INPUT_DEPLOYMENT_TAG: ${{ inputs.deployment-tag }}
      run: python -m ferry_action.deploy_apigw
```

Note: Same structure as the other composite actions. OIDC auth first, then uv install, then run the Python module. Region is handled by configure-aws-credentials.
