GREEN: Create `action/src/ferry_action/deploy_stepfunctions.py` following the same structure as `deploy.py`:

```python
"""Step Functions deployment with envsubst and content-hash skip.

Deploys a state machine definition:
1. Reads definition file from source_dir.
2. Performs envsubst (${ACCOUNT_ID}, ${AWS_REGION}).
3. Checks content hash against ferry:content-hash tag (skip if unchanged).
4. Calls update_state_machine with publish=True.
5. Updates the content hash tag.

Invoked as: python -m ferry_action.deploy_stepfunctions
"""
```

Functions:
- `should_skip_deploy(sfn_client, state_machine_arn: str, new_hash: str) -> bool`: Call `list_tags_for_resource`, extract `ferry:content-hash` via `get_content_hash_tag`, compare with `new_hash`.
- `deploy_step_function(sfn_client, state_machine_arn: str, definition: str, deployment_tag: str) -> dict`: Call `update_state_machine(stateMachineArn=arn, definition=definition, publish=True, versionDescription=f"Deployed by Ferry: {deployment_tag}")`. Then `tag_resource` with `ferry:content-hash` set to `compute_content_hash(definition)`. Return `{"version_arn": response.get("stateMachineVersionArn", ""), "state_machine_arn": state_machine_arn, "skipped": False}`.
- `main()`:
  1. Read env vars: INPUT_RESOURCE_NAME, INPUT_STATE_MACHINE_NAME, INPUT_DEFINITION_FILE, INPUT_SOURCE_DIR, INPUT_DEPLOYMENT_TAG, INPUT_TRIGGER_SHA.
  2. Create STS client, get account_id via `get_caller_identity()["Account"]`.
  3. Get region from `AWS_DEFAULT_REGION` or `AWS_REGION` env var (default "us-east-1").
  4. Construct ARN: `arn:aws:states:{region}:{account_id}:stateMachine:{state_machine_name}`.
  5. Read definition file from `Path(source_dir) / definition_file`.
  6. Apply `envsubst(definition_content, account_id, region)`.
  7. Compute content hash of substituted definition.
  8. Check skip (call `should_skip_deploy`).
  9. If skip: set GHA outputs (skipped=true), write skip summary, return.
  10. Deploy (call `deploy_step_function`).
  11. Set GHA outputs (skipped=false, version-arn=...).
  12. Write GHA summary table (same format as Lambda deploy: resource name, state machine name, definition file, version ARN, status, deployment tag).
  13. Error handling: catch `ClientError` with hint dict for `StateMachineDoesNotExist`, `AccessDeniedException`, `InvalidDefinition`. Same pattern as `deploy.py`.

All within `gha.begin_group` / `gha.end_group`.

Run tests -- they MUST pass.

Create `action/deploy-stepfunctions/action.yml` following the exact same pattern as `action/deploy/action.yml`:

```yaml
name: "Ferry Deploy Step Functions"
description: "Deploy Step Functions state machine definition"

inputs:
  resource-name:
    description: "Name of the Step Functions resource"
    required: true
  state-machine-name:
    description: "State machine name in AWS"
    required: true
  definition-file:
    description: "Definition file name relative to source-dir"
    required: true
  source-dir:
    description: "Source directory containing the definition file"
    required: true
  trigger-sha:
    description: "Git SHA that triggered the deployment"
    required: true
  deployment-tag:
    description: "Deployment tag (e.g. pr-42)"
    required: true
  aws-role-arn:
    description: "IAM role ARN for OIDC authentication"
    required: true
  aws-region:
    description: "AWS region"
    required: false
    default: "us-east-1"

outputs:
  skipped:
    description: "Whether deploy was skipped (definition unchanged)"
    value: ${{ steps.deploy.outputs.skipped }}
  version-arn:
    description: "Published state machine version ARN"
    value: ${{ steps.deploy.outputs.version-arn }}

runs:
  using: composite
  steps:
    - name: Configure AWS credentials via OIDC
      uses: aws-actions/configure-aws-credentials@v4
      with:
        role-to-assume: ${{ inputs.aws-role-arn }}
        aws-region: ${{ inputs.aws-region }}

    - name: Install uv
      uses: astral-sh/setup-uv@v4

    - name: Install ferry-action package
      shell: bash
      run: uv pip install --system "${{ github.action_path }}/.."

    - name: Deploy Step Functions
      id: deploy
      shell: bash
      env:
        INPUT_RESOURCE_NAME: ${{ inputs.resource-name }}
        INPUT_STATE_MACHINE_NAME: ${{ inputs.state-machine-name }}
        INPUT_DEFINITION_FILE: ${{ inputs.definition-file }}
        INPUT_SOURCE_DIR: ${{ inputs.source-dir }}
        INPUT_TRIGGER_SHA: ${{ inputs.trigger-sha }}
        INPUT_DEPLOYMENT_TAG: ${{ inputs.deployment-tag }}
      run: python -m ferry_action.deploy_stepfunctions
```

Note: The action.yml maps GHA inputs to INPUT_* env vars, matching the pattern from deploy/action.yml. The `aws-region` is handled by configure-aws-credentials (sets AWS_DEFAULT_REGION and AWS_REGION env vars), so deploy_stepfunctions.py reads it from there.
