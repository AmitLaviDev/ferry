Create `scripts/bootstrap.sh` — an idempotent bash script that orchestrates the full Phase 11 bootstrap sequence.

**Script structure:**

```
#!/usr/bin/env bash
set -euo pipefail
```

**Constants:**
- `SCRIPT_DIR` = resolved script directory
- `REPO_ROOT` = parent of SCRIPT_DIR
- `BUCKET_NAME` = "ferry-terraform-state"
- `ECR_REPO` = "lambda-ferry-backend"
- `AWS_REGION` = "us-east-1"
- `PLACEHOLDER_TAG` = "placeholder"
- `BACKEND_DIR` = "$REPO_ROOT/iac/global/aws/backend"
- `ECR_DIR` = "$REPO_ROOT/iac/global/aws/ecr"

**Logging helpers:**
- `log_step()` — prints step header with `==>` prefix and bold (tput or ANSI if available)
- `log_info()` — prints info message
- `log_skip()` — prints skip message (already done)
- `log_error()` — prints error to stderr

**Prerequisite check function:**
- Verify `terraform` is installed (command -v terraform)
- Verify `aws` CLI is installed (command -v aws)
- Verify `docker` is installed (command -v docker)
- Verify AWS credentials are valid: `aws sts get-caller-identity` (fail with helpful message if not configured)
- Print account ID and caller ARN for confirmation

**Step 1: Bootstrap S3 state backend (`step_backend`):**
1. Check if S3 bucket already exists: `aws s3api head-bucket --bucket "$BUCKET_NAME" 2>/dev/null`
2. If bucket does NOT exist:
   a. `terraform -chdir="$BACKEND_DIR" init -backend=false` (skip S3 backend since it doesn't exist yet)
   b. `terraform -chdir="$BACKEND_DIR" apply -auto-approve`
   c. Log success
3. Check if state is already remote by examining `.terraform/terraform.tfstate` in BACKEND_DIR — if backend type is already "s3", skip migration
4. If state is local:
   a. `terraform -chdir="$BACKEND_DIR" init -migrate-state -force-copy` (migrates local state to S3)
   b. Log migration success
5. If bucket already existed AND state is remote, log skip

**Key detail for step 1:** The providers.tf file from Plan 01 has the `backend "s3"` block present. The trick is:
- First `terraform init -backend=false` ignores the backend block entirely (treats it as local)
- After apply creates the bucket, `terraform init -migrate-state -force-copy` picks up the backend block and migrates

**Step 2: Create ECR repository (`step_ecr`):**
1. Check if ECR repo exists: `aws ecr describe-repositories --repository-names "$ECR_REPO" --region "$AWS_REGION" 2>/dev/null`
2. If not exists:
   a. `terraform -chdir="$ECR_DIR" init`
   b. `terraform -chdir="$ECR_DIR" apply -auto-approve`
3. If exists, log skip

**Step 3: Build and push placeholder image (`step_placeholder`):**
1. Check if placeholder image already exists: `aws ecr describe-images --repository-name "$ECR_REPO" --image-ids imageTag="$PLACEHOLDER_TAG" --region "$AWS_REGION" 2>/dev/null`
2. If not exists:
   a. Get account ID: `aws sts get-caller-identity --query Account --output text`
   b. Construct ECR URI: `${ACCOUNT_ID}.dkr.ecr.${AWS_REGION}.amazonaws.com/${ECR_REPO}`
   c. ECR login: `aws ecr get-login-password --region "$AWS_REGION" | docker login --username AWS --password-stdin "${ACCOUNT_ID}.dkr.ecr.${AWS_REGION}.amazonaws.com"`
   d. Build: `docker build --platform linux/arm64 -t "${ECR_URI}:${PLACEHOLDER_TAG}" "$ECR_DIR/placeholder"`
   e. Push: `docker push "${ECR_URI}:${PLACEHOLDER_TAG}"`
3. If exists, log skip

**Main function:**
```bash
main() {
  log_step "Ferry Bootstrap"
  check_prerequisites
  step_backend
  step_ecr
  step_placeholder
  log_step "Bootstrap complete"
  # Print summary of what was created/skipped
}

main "$@"
```

**Error handling:**
- `set -euo pipefail` at top
- `trap` to catch errors and print which step failed
- Each step function should have clear error context
