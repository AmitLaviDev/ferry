<objective>
Replace the push handler's "is default branch?" gate with environment-based gating. After this plan, pushes to environment-mapped branches with `auto_deploy: true` trigger deploys with the environment name in the payload, while all other pushes (unmapped branches, auto_deploy: false, no environments configured, branch deletions, tag pushes) produce zero Ferry activity.

Purpose: Enables auto-deploy on merge to any mapped branch (not just default), with environment context flowing through to the dispatch payload. This is the core push-path behavior for v2.0 PR Integration.

Output: Refactored push handler in handler.py, 7 integration tests in test_handler_push_env.py, updated ENV-03 in REQUIREMENTS.md.

```python
def resolve_environment(
    config: FerryConfig,
    base_branch: str,
) -> EnvironmentMapping | None:
    """Find the environment mapping for a target branch."""
    for env in config.environments:
        if env.branch == base_branch:
            return env
    return None
```
