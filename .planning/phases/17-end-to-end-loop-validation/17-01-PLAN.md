Fix the known find_open_prs 403 bug before starting the E2E validation loop.

Purpose: The handler calls find_open_prs on every push event. This would block the entire E2E loop. Same issue exists in find_merged_pr.
Output: Patched runs.py with graceful 403/error handling, new tests proving the fix.
</objective>

Pattern:
```python

        "pr_lookup_failed",

        repo=repo,
        sha=sha[:7],
    )

```
