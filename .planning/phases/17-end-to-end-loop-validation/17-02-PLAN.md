  <action>
Infrastructure is verified healthy from Task 1. Execute the actual E2E push-to-deploy loop:

**Step A: Push a code change to test repo**
```bash
cd /path/to/ferry-test-app
# Edit lambdas/hello-world/main.py to change the greeting message
# e.g., change "hello from ferry-test" to "hello from ferry-test v2"
git add lambdas/hello-world/main.py
git commit -m "feat: update greeting to v2"
git push origin main
```

**Step B: Verify Ferry received the webhook** (within 30 seconds)
- Check CloudWatch: `aws logs tail /aws/lambda/ferry-backend --since 5m --format short`
- Look for: `webhook_event_received`, `changes_detected`, `dispatches_triggered`
- If NO logs appear: check GitHub App webhook deliveries page for errors

**Step C: Verify workflow_dispatch was triggered** (within 1 minute)
- `gh run list -R AmitLaviDev/ferry-test-app --limit 3`
- Should show a new workflow run (status: in_progress or completed)
- If NO run: check CloudWatch for dispatch errors, check ferry-test-app Actions tab

**Step D: Verify GHA workflow completes** (wait up to 5 minutes)
- `gh run view -R AmitLaviDev/ferry-test-app RUN_ID`
- All jobs should complete successfully
- If FAILED: `gh run view -R AmitLaviDev/ferry-test-app RUN_ID --log-failed` for error details

**Step E: Verify Lambda was deployed**
- `aws lambda get-function --function-name ferry-test-hello-world --query 'Configuration.LastModified'`
- Should show a recent timestamp (within last few minutes)

**Step F: Invoke the deployed Lambda**
```bash
aws lambda invoke --function-name ferry-test-hello-world --qualifier live --payload '{}' /tmp/ferry-test-output.json && cat /tmp/ferry-test-output.json
```
- Should return the updated greeting message ("hello from ferry-test v2")

**If any step fails:**
1. Diagnose from CloudWatch logs (webhook processing) or GHA logs (build/deploy)
2. Fix the bug in the ferry repo with an atomic commit (one fix per commit per user decision)
3. Push to ferry/main (triggers self-deploy -- no manual Lambda updates per user decision)
4. Wait for self-deploy to complete
5. Retry from the failed step
