<feature>
  <name>Batched Dispatch with Payload-Size Fallback</name>
  <files>backend/src/ferry_backend/dispatch/trigger.py, tests/test_backend/test_dispatch_trigger.py</files>
  <behavior>
    **Batched dispatch (happy path - DISP-01):**
    - Multi-type push (e.g., lambda + step_function): exactly 1 HTTP POST to GitHub API, payload is BatchedDispatchPayload v2 with both types populated
    - Single-type push (e.g., lambda only): exactly 1 HTTP POST, payload is BatchedDispatchPayload v2 with only lambdas populated
    - All-three-types push: exactly 1 HTTP POST, payload v2 with all three lists populated
    - Empty affected list: 0 HTTP POSTs, returns []
    - Return value: list[dict] with one {"type": str, "status": int, "workflow": str} entry per resource type (expanded from single API call)
    - Batched payload validates as BatchedDispatchPayload with v=2

    **Fallback on oversized payload (DISP-03):**
    - When serialized batched payload exceeds 65,535 chars: falls back to per-type v1 dispatch
    - Fallback produces N HTTP POSTs (one per type), each with v1 DispatchPayload
    - Fallback log event: dispatch_fallback_to_per_type with payload_size field
    - Fallback returns same list[dict] shape as batched path

    **Logging:**
    - Batched dispatch: dispatch_triggered events with mode="batched"
    - Fallback dispatch: dispatch_triggered events with mode="per_type"

    **Preserved behavior:**
    - build_deployment_tag() unchanged
    - _build_resource() unchanged
    - Existing empty-list test still passes
  </behavior>
  <implementation>
    1. Extract existing per-type loop (lines 141-184) into _dispatch_per_type() private helper with same signature minus `affected` plus `grouped` dict
    2. Add _TYPE_TO_FIELD mapping dict: {"lambda": "lambdas", "step_function": "step_functions", "api_gateway": "api_gateways"}
    3. Rewrite trigger_dispatches() main body:
       - Group by type (same as current)
       - Build typed resource lists using _TYPE_TO_FIELD and _build_resource()
       - Construct BatchedDispatchPayload with **typed_resources spread
       - Serialize with model_dump_json(), store in variable (use for both size check AND dispatch)
       - If len(payload_json) > _MAX_PAYLOAD_SIZE: log warning, call _dispatch_per_type()
       - Else: single POST, expand result into per-type entries with mode="batched" logging
    4. Update imports: add BatchedDispatchPayload, BATCHED_SCHEMA_VERSION from ferry-utils
    5. Update module docstring to reflect batched dispatch (not per-type)
  </implementation>
</feature>
