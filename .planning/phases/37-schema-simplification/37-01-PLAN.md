**backend/src/ferry_backend/config/schema.py:**

1. In `LambdaConfig`: Remove the `function_name: str | None = None` field entirely. Replace the `@model_validator(mode="after") set_function_name_default` with a `@model_validator(mode="before")` named `handle_deprecated_function_name` that:
   - If `isinstance(data, dict)` and `"function_name"` is in `data`:
     - If `"name"` is NOT in `data`: set `data["name"] = data.pop("function_name")`
     - Else: just `data.pop("function_name")` (name wins, silently drop the old field)
   - Return `data`
   - This runs BEFORE Pydantic field validation, so `extra="forbid"` never sees `function_name`.

2. In `StepFunctionConfig`: Remove the `state_machine_name: str` field. Add a `@model_validator(mode="before")` named `handle_deprecated_state_machine_name` that:
   - If `isinstance(data, dict)` and `"state_machine_name"` is in `data`:
     - If `"name"` is NOT in `data`: set `data["name"] = data.pop("state_machine_name")`
     - If `"name"` IS in `data` AND values differ: set `data["name"] = data.pop("state_machine_name")` (state_machine_name is the AWS name, it should win per research Pitfall 5)
     - Else: just `data.pop("state_machine_name")`
   - Return `data`
   - IMPORTANT: `StepFunctionConfig` currently has `state_machine_name` as a required field with no default. After removal, `name` (already required) is the only name field.

3. `ApiGatewayConfig`: NO CHANGES. `rest_api_id` and `stage_name` are AWS identifiers, not names.

**utils/src/ferry_utils/models/dispatch.py:**

4. In `LambdaResource`: Remove the `function_name: str` field. The model now has: `resource_type`, `name`, `source`, `ecr`, `runtime`.

5. In `StepFunctionResource`: Remove the `state_machine_name: str` field. The model now has: `resource_type`, `name`, `source`, `definition_file`.

6. `ApiGatewayResource`: NO CHANGES.

**backend/src/ferry_backend/dispatch/trigger.py:**

7. In `_build_resource()`:
   - Lambda case: Remove `function_name=lam.function_name` from `LambdaResource()` constructor. It already passes `name=name`.
   - Step Function case: Remove `state_machine_name=sf.state_machine_name` from `StepFunctionResource()` constructor. It already passes `name=name`.
   - Update the docstring to remove references to `function_name`.
