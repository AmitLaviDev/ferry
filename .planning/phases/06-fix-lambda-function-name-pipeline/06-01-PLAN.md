Wire `function_name` through the full Lambda dispatch pipeline: LambdaResource model -> _build_resource -> parse_payload matrix -> deploy.py, closing the DEPLOY-01 integration break where function_name is dropped between LambdaConfig (backend) and deploy.py (action).

Purpose: Without this fix, the Lambda deploy action never receives the function_name from ferry.yaml, meaning deploy.py cannot target the correct AWS Lambda function. This is the critical data-plumbing gap preventing end-to-end Lambda deployment.

Output: All 4 source files updated, all test files updated to include function_name, full test suite passing.
</objective>
