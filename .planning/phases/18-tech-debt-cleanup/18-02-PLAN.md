  <files>iac/aws/staging/shared/data.tf</files>
  <action>
In `iac/aws/staging/shared/data.tf`, find the `gha_self_deploy_lambda` policy document (lines 171-183).

Add `"lambda:GetFunctionConfiguration"` to the actions list, after `"lambda:GetFunction"`. The updated block should be:

```hcl
data "aws_iam_policy_document" "gha_self_deploy_lambda" {
  statement {
    sid    = "LambdaUpdate"
    effect = "Allow"
    actions = [
      "lambda:UpdateFunctionCode",
      "lambda:GetFunction",
      "lambda:GetFunctionConfiguration",
    ]
    resources = [
      "arn:aws:lambda:${data.aws_region.current.id}:${data.aws_caller_identity.current.account_id}:function:ferry-backend",
    ]
  }
}
```

This matches the pattern already used in `iac/test-env/data.tf` for the test deploy role. The `GetFunctionConfiguration` permission was discovered as needed during v1.2 E2E testing (commit adb3e34) but was only added to the test-env role, not the self-deploy role.

NOTE: This Terraform change will require a `terraform apply` in `iac/aws/staging/shared/` to take effect. The executor should note this in the summary but NOT attempt to run terraform apply.
