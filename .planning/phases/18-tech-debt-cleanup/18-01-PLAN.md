**Implementation** (in action/src/ferry_action/deploy.py):

In the `except ClientError` block (lines 229-248):

1. Remove the `raw = f"[{error_code}] {error_message}"` line and the `(raw: {raw})` from the `gha.error()` call. The `format_error_detail` function already handles debug output via `FERRY_DEBUG`.

2. Replace the simple `AccessDeniedException` hint with logic that inspects `error_message`:
   - If message contains "not authorized to perform" or "AccessDenied" -> hint about the deploy role (caller) lacking the specific permission.
   - If message contains "cannot be assumed" or "role defined for the function" -> hint about the Lambda's execution role configuration.
   - Otherwise (generic) -> combined hint: "Either the deploy role lacks permissions for '{function_name}', or the Lambda's execution role cannot be assumed. Check IAM policies for both roles."

3. Keep the `report_check_run` call with the hint (not the raw error).

The updated error handling block should look like:
```python
except ClientError as exc:
    error_code = exc.response["Error"]["Code"]
    error_message = exc.response["Error"].get("Message", "")
    if error_code == "AccessDeniedException":
        msg_lower = error_message.lower()
        if "not authorized to perform" in msg_lower:
            hint = (
                f"Deploy role lacks permissions for '{function_name}'. "
                f"Check that the IAM role has lambda:UpdateFunctionCode, "
                f"lambda:GetFunction, and lambda:GetFunctionConfiguration."
            )
        elif "cannot be assumed" in msg_lower or "role defined for the function" in msg_lower:
            hint = (
                f"Lambda '{function_name}' execution role cannot be assumed. "
                f"Check the Lambda's execution role trust policy and permissions."
            )
        else:
            hint = (
                f"Permission denied for '{function_name}'. "
                f"Either the deploy role lacks Lambda permissions, "
                f"or the function's execution role cannot be assumed. "
                f"Check IAM policies for both roles."
            )
    elif error_code == "ResourceNotFoundException":
        hint = (
            f"Lambda function '{function_name}' not found. "
            f"Check ferry.yaml function_name or verify the "
            f"Lambda exists in the target account."
        )
    else:
        hint = f"Deploy failed: [{error_code}] {error_message}"

    gha.error(
        format_error_detail(exc, f"Deploy failed for {resource_name}: {hint}")
    )
    report_check_run(resource_name, "deploy", "failure", hint, trigger_sha)
    sys.exit(1)
```

**In action/src/ferry_action/build.py**, modify the `ecr_login` function's `docker login` subprocess call (lines 110-122).

The current call does NOT use `capture_output=True`, which means Docker's warning "WARNING! Your password will be stored unencrypted in /home/runner/.docker/config.json" prints to stderr in GHA logs.

Change the `docker login` subprocess.run call to add `capture_output=True` so stderr is captured and suppressed:

```python
    result = subprocess.run(
        [
            "docker",
            "login",
            "--username",
            "AWS",
            "--password-stdin",
            registry,
        ],
        input=password_result.stdout.strip(),
        capture_output=True,
        text=True,
        check=True,
    )
```

Note: `capture_output=True` captures both stdout and stderr. Since `check=True` is set, any non-zero exit code will still raise `CalledProcessError`. The warning is cosmetic and safe to suppress.

**In tests/test_action/test_build.py**, update the `TestEcrLogin` class:
