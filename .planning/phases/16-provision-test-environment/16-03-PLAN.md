**Step 1: Push a placeholder image to ECR (chicken-and-egg)**

Before `terraform apply`, we need a placeholder image URI for the Lambda. The simplest approach:
1. Create the ECR repo first by running terraform apply with the Lambda module commented out (or use `-target`):
   ```bash
   cd iac/test-env
   terraform init
   terraform apply -target=module.ecr_test -auto-approve
   ```
2. Pull and push a placeholder image:
   ```bash
   ACCOUNT_ID=$(aws sts get-caller-identity --query Account --output text)
   ECR_URI="${ACCOUNT_ID}.dkr.ecr.us-east-1.amazonaws.com"
   aws ecr get-login-password --region us-east-1 | docker login --username AWS --password-stdin "$ECR_URI"
   docker pull public.ecr.aws/lambda/python:3.12
   docker tag public.ecr.aws/lambda/python:3.12 "${ECR_URI}/ferry-test/hello-world:placeholder"
   docker push "${ECR_URI}/ferry-test/hello-world:placeholder"
   ```
3. Now apply the full project with the placeholder URI:
   ```bash
   terraform apply -var="lambda_placeholder_image_uri=${ECR_URI}/ferry-test/hello-world:placeholder" -auto-approve
   ```

Alternatively, set `lambda_placeholder_image_uri` to `public.ecr.aws/lambda/python:3.12` directly if the Lambda module accepts public ECR URIs (it should -- Lambda supports cross-account ECR). This is simpler. Try this first:
```bash
cd iac/test-env
terraform init
terraform apply -var="lambda_placeholder_image_uri=public.ecr.aws/lambda/python:3.12" -auto-approve
```
