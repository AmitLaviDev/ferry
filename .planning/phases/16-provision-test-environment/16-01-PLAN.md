Create the `iac/test-env/` directory and three files following existing IaC conventions:

**providers.tf:**
- `terraform` block with `required_version = "~> 1.12.0"`, S3 backend pointing to `ferry-global-terraform-state` bucket with key `test-env/terraform.tfstate`, region `us-east-1`, `use_lockfile = true`

- `provider "aws"` with `region = var.region` and `default_tags` for `ManagedBy = "terraform"` and `Project = "ferry-test"`
- Comment: "No assume_role -- uses ambient credentials (same pattern as global TF projects)"

**variables.tf:**
- `region` (string, default `"us-east-1"`)
- `github_repo` (string, default `"ferry-test-app"`, description: "GitHub repo name for OIDC trust policy")
- `github_owner` (string, default `"AmitLaviDev"`, description: "GitHub owner for OIDC trust policy")
- `ecr_repository_name` (string, default `"ferry-test/hello-world"`, description: "ECR repo name for test Lambda")
- `lambda_function_name` (string, default `"ferry-test-hello-world"`, description: "Lambda function name for test Lambda")
- `lambda_placeholder_image_uri` (string, no default, description: "Placeholder image URI from ECR -- set after first ECR push or use public.ecr.aws/lambda/python:3.12")

**data.tf:**
- `data "aws_caller_identity" "current" {}` and `data "aws_region" "current" {}`
- `data "terraform_remote_state" "oidc"` referencing the OIDC provider state at `global/cloud/aws/oidc/terraform.tfstate` (same pattern as `iac/aws/staging/shared/data.tf`)
- `data "aws_iam_policy_document" "test_deploy_assume_role"` -- OIDC trust policy with:
  - Action: `sts:AssumeRoleWithWebIdentity`
  - Principal: Federated, `data.terraform_remote_state.oidc.outputs.oidc_provider_arn`
  - Condition `StringEquals` on `aud` = `"sts.amazonaws.com"`
  - Condition `StringLike` on `sub` = `"repo:${var.github_owner}/${var.github_repo}:*"`
- `data "aws_iam_policy_document" "test_ecr_push"` -- ECR push permissions (same actions as `gha_dispatch_ecr` in shared/data.tf) scoped to `arn:aws:ecr:${data.aws_region.current.id}:${data.aws_caller_identity.current.account_id}:repository/${var.ecr_repository_name}`
- `data "aws_iam_policy_document" "test_ecr_auth"` -- `ecr:GetAuthorizationToken` on `"*"`
- `data "aws_iam_policy_document" "test_lambda_deploy"` -- Lambda update permissions: `lambda:UpdateFunctionCode`, `lambda:GetFunction`, `lambda:PublishVersion`, `lambda:UpdateAlias`, `lambda:CreateAlias`, `lambda:GetAlias` scoped to `arn:aws:lambda:${data.aws_region.current.id}:${data.aws_caller_identity.current.account_id}:function:${var.lambda_function_name}` (Note: ferry-action needs PublishVersion + UpdateAlias/CreateAlias/GetAlias beyond just UpdateFunctionCode+GetFunction -- check deploy.py for the full set of API calls)

  <action>
**main.tf:**

Create three resource groups:

1. **ECR Repository** -- use the `terraform-aws-modules/ecr/aws` module (version `2.4.0`, same as `iac/global/cloud/aws/ecr/main.tf`):
   - `repository_name = var.ecr_repository_name`
   - `repository_image_tag_mutability = "MUTABLE"`
   - `repository_force_delete = false`
   - `repository_image_scan_on_push = true`
   - Lifecycle policy: keep last 10 images (same JSON as existing ECR project)
   - Tag: `Name = var.ecr_repository_name`

2. **IAM Role + Policies** -- follow the pattern from `iac/aws/staging/shared/iam.tf`:
   - `aws_iam_role "test_deploy"` with `name = "ferry-test-deploy"`, `assume_role_policy = data.aws_iam_policy_document.test_deploy_assume_role.json`, tag `Name = "ferry-test-deploy"`
   - `aws_iam_policy "test_ecr_auth"` with `name = "ferry-test-ecr-auth"` using `data.aws_iam_policy_document.test_ecr_auth.json`
   - `aws_iam_policy "test_ecr_push"` with `name = "ferry-test-ecr-push"` using `data.aws_iam_policy_document.test_ecr_push.json`
   - `aws_iam_policy "test_lambda_deploy"` with `name = "ferry-test-lambda-deploy"` using `data.aws_iam_policy_document.test_lambda_deploy.json`
   - Three `aws_iam_role_policy_attachment` resources attaching all three policies to the role

3. **Lambda Function** -- use the `terraform-aws-modules/lambda/aws` module (version `8.7.0`, same as ferry_backend):
   - `function_name = var.lambda_function_name`
   - `description = "Ferry test hello-world Lambda -- pre-created for E2E validation"`
   - `create_role = false` -- we need a basic execution role. Use `aws_iam_role.test_lambda_execution.arn` (create a simple role below)
   - `create_package = false`, `package_type = "Image"`, `image_uri = var.lambda_placeholder_image_uri`, `ignore_source_code_hash = true`
   - `memory_size = 128`, `timeout = 30`
   - `architectures = ["x86_64"]` -- test Lambda uses x86_64 for compatibility with standard Python base images
   - `create_lambda_function_url = false` -- no Function URL needed for test Lambda
   - Tag: `Name = var.lambda_function_name`
   - Add `lifecycle { ignore_changes = [image_uri] }` -- same pattern as ferry_backend, TF owns infra, Ferry owns code

4. **Lambda Execution Role** (minimal, just for the test Lambda to run):
   - `aws_iam_role "test_lambda_execution"` with Lambda assume role trust policy
   - Add a `data "aws_iam_policy_document" "test_lambda_assume_role"` in data.tf for the Lambda service principal
   - Attach `arn:aws:iam::aws:policy/service-role/AWSLambdaBasicExecutionRole` managed policy

**outputs.tf:**
- `test_deploy_role_arn` -- the IAM role ARN for GHA runners (this gets set as `AWS_ROLE_ARN` secret in the test repo)
- `ecr_repository_url` -- full ECR repo URL for pushing images
- `lambda_function_name` -- Lambda function name for verification
- `lambda_function_arn` -- Lambda ARN
