**.github/workflows/ferry-lambdas.yml:**

Adapt from `docs/lambdas.md` workflow template but with critical differences for the test repo:
- The test repo does NOT contain ferry-action source code -- it must reference the ferry repo's actions via `AmitLaviDev/ferry@main` (NOT `./action/setup`, `./action/build`, `./action/deploy`)
- This is the key difference: the ferry repo uses `./action/*` (local), but consumer repos use `{owner}/{repo}@{ref}` (remote)

```yaml
name: Ferry Lambdas

on:
  workflow_dispatch:
    inputs:
      payload:
        description: "Ferry dispatch payload (JSON)"
        required: true

permissions:
  id-token: write
  contents: read
  checks: write

jobs:
  setup:
    runs-on: ubuntu-latest
    outputs:
      matrix: ${{ steps.parse.outputs.matrix }}
    steps:
      - uses: actions/checkout@v4
      - name: Parse Ferry payload
        id: parse
        uses: AmitLaviDev/ferry@main
        with:
          path: action/setup
          payload: ${{ inputs.payload }}

  deploy:
    needs: setup
    runs-on: ubuntu-latest
    strategy:
      matrix: ${{ fromJson(needs.setup.outputs.matrix) }}
      fail-fast: false
    steps:
      - uses: actions/checkout@v4

      - name: Build container
        id: build
        uses: AmitLaviDev/ferry@main
        with:
          path: action/build
          resource-name: ${{ matrix.name }}
          source-dir: ${{ matrix.source }}
          ecr-repo: ${{ matrix.ecr }}
          aws-role-arn: ${{ secrets.AWS_ROLE_ARN }}
          aws-region: us-east-1
          trigger-sha: ${{ matrix.trigger_sha }}
          deployment-tag: ${{ matrix.deployment_tag }}
          runtime: ${{ matrix.runtime }}

      - name: Deploy Lambda
        uses: AmitLaviDev/ferry@main
        with:
          path: action/deploy
          resource-name: ${{ matrix.name }}
          function-name: ${{ matrix.function_name }}
          image-uri: ${{ steps.build.outputs.image-uri }}
          image-digest: ${{ steps.build.outputs.image-digest }}
          deployment-tag: ${{ matrix.deployment_tag }}
          trigger-sha: ${{ matrix.trigger_sha }}
          aws-role-arn: ${{ secrets.AWS_ROLE_ARN }}
          aws-region: us-east-1
          github-token: ${{ github.token }}
```

IMPORTANT: The `uses:` syntax for composite actions from another repo requires careful verification. Check how GitHub Actions resolves composite actions from external repos. The syntax is typically `{owner}/{repo}/{path}@{ref}` or `{owner}/{repo}@{ref}` with a `path` input. Since ferry's composite actions are at `action/setup/action.yml`, `action/build/action.yml`, `action/deploy/action.yml`, the correct `uses:` would be:
- `uses: AmitLaviDev/ferry/action/setup@main`
- `uses: AmitLaviDev/ferry/action/build@main`
- `uses: AmitLaviDev/ferry/action/deploy@main`

Use the `{owner}/{repo}/{path}@{ref}` format (NOT the `path:` input format, which is for reusable workflows, not composite actions). Remove the `with: path:` lines and use the correct `uses:` format above.
