**data.tf** — Data sources and all `data.aws_iam_policy_document` blocks:

Data sources:
- `data "aws_caller_identity" "current" {}`
- `data "aws_region" "current" {}`
- `data "terraform_remote_state" "oidc"` referencing `bucket = "ferry-global-terraform-state"`, `key = "global/aws/oidc/terraform.tfstate"`, `region = "us-east-1"`

Trust policy documents (assume role):
- `data "aws_iam_policy_document" "lambda_assume_role"` — Allow `sts:AssumeRole` from `Service: lambda.amazonaws.com`

Permission policy documents (what roles can DO):
- `data "aws_iam_policy_document" "lambda_dynamodb"` — Allow `dynamodb:GetItem`, `PutItem`, `DeleteItem`, `Query` on `arn:aws:dynamodb:${data.aws_region.current.name}:${data.aws_caller_identity.current.account_id}:table/ferry-*`
- `data "aws_iam_policy_document" "lambda_secrets"` — Allow `secretsmanager:GetSecretValue` on `arn:aws:secretsmanager:${data.aws_region.current.name}:${data.aws_caller_identity.current.account_id}:secret:ferry/*`
- `data "aws_iam_policy_document" "lambda_logs"` — Allow `logs:CreateLogStream`, `logs:PutLogEvents` on `arn:aws:logs:${data.aws_region.current.name}:${data.aws_caller_identity.current.account_id}:log-group:/aws/lambda/ferry-*:*`
- `data "aws_iam_policy_document" "gha_ecr_auth"` — Allow `ecr:GetAuthorizationToken` on `Resource = "*"` (account-level, cannot be scoped)
- `data "aws_iam_policy_document" "gha_self_deploy_ecr"` — Allow `ecr:BatchCheckLayerAvailability`, `BatchGetImage`, `CompleteLayerUpload`, `GetDownloadUrlForLayer`, `InitiateLayerUpload`, `PutImage`, `UploadLayerPart` on `arn:aws:ecr:${data.aws_region.current.name}:${data.aws_caller_identity.current.account_id}:repository/lambda-ferry-backend`
- `data "aws_iam_policy_document" "gha_self_deploy_lambda"` — Allow `lambda:UpdateFunctionCode`, `lambda:GetFunction` on `arn:aws:lambda:${data.aws_region.current.name}:${data.aws_caller_identity.current.account_id}:function:ferry-backend`
- `data "aws_iam_policy_document" "gha_dispatch_ecr"` — Allow same ECR push actions but on `repository/ferry/*` (broader scope for customer repo dispatches, matching research recommendation for dispatch role)
- `data "aws_iam_policy_document" "gha_dispatch_lambda"` — Allow `lambda:UpdateFunctionCode`, `lambda:GetFunction` on `function:ferry-*` (broader scope)

Each policy document should have a descriptive `sid` on the statement (e.g., `sid = "DynamoDBAccess"`).

**oidc.tf** — OIDC assume-role trust policy documents for the two GHA roles. Separate file because these are the OIDC-specific trust bindings:

- `data "aws_iam_policy_document" "gha_self_deploy_assume_role"` — Allow `sts:AssumeRoleWithWebIdentity` from `Federated: data.terraform_remote_state.oidc.outputs.oidc_provider_arn`, with two conditions:
  - `StringEquals` on `token.actions.githubusercontent.com:aud` = `["sts.amazonaws.com"]`
  - `StringLike` on `token.actions.githubusercontent.com:sub` = `["repo:${var.github_org}/${var.github_repo}:*"]` (scoped to ferry repo)

- `data "aws_iam_policy_document" "gha_dispatch_assume_role"` — Same structure but with:
  - `StringLike` on `token.actions.githubusercontent.com:sub` = `["repo:${var.github_org}/*:*"]` (org-wide scope for customer repos, per research recommendation for dispatch role)
