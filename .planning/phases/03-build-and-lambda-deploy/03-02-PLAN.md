<feature>
  <name>Magic Dockerfile and Build Pipeline</name>
  <files>action/Dockerfile, action/src/ferry_action/build.py, tests/test_action/test_build.py</files>
  <behavior>
    Build module constructs and executes Docker commands for Lambda container builds:

    get_ecr_uri(aws_account_id, aws_region, ecr_repo) -> "{account_id}.dkr.ecr.{region}.amazonaws.com/{ecr_repo}"

    ecr_login(ecr_uri) -> runs: aws ecr get-login-password | docker login --username AWS --password-stdin {registry}
    - Registry is the URI up to the repo name (just the account.dkr.ecr.region.amazonaws.com part)

    build_image(source_dir, ecr_uri, deployment_tag, runtime, dockerfile_path, github_token="") -> image_tag
    - Strips "python" prefix from runtime string (e.g., "python3.12" -> "3.12") for PYTHON_VERSION build arg
    - Constructs: docker build --build-arg PYTHON_VERSION={version} --tag {ecr_uri}:{deployment_tag} --file {dockerfile_path} {source_dir}
    - If github_token is non-empty, adds: --secret id=github_token,env=GITHUB_TOKEN
    - Runs command via subprocess.run with check=True
    - Wraps Docker build output in GHA collapsible group via gha.group()
    - On failure, calls gha.failure_hint() on stderr and logs hint if available
    - Returns the full image tag "{ecr_uri}:{deployment_tag}"

    push_image(image_tag) -> digest (str)
    - Runs: docker push {image_tag}
    - Wraps push output in GHA collapsible group
    - After push, runs: docker inspect --format='{{index .RepoDigests 0}}' {image_tag}
    - Extracts sha256 digest from the RepoDigest string (format: "repo@sha256:abc...")
    - Returns the digest string "sha256:abc..."

    Cases:
    - build_image("services/order", "123.dkr.ecr.us-east-1.amazonaws.com/ferry/order", "pr-42", "python3.12", "/path/Dockerfile") -> constructs correct docker build command without secret
    - build_image(..., github_token="ghp_abc") -> includes --secret flag
    - build_image with runtime "python3.14" -> --build-arg PYTHON_VERSION=3.14
    - get_ecr_uri("123456789012", "us-east-1", "ferry/order") -> "123456789012.dkr.ecr.us-east-1.amazonaws.com/ferry/order"
    - push_image returns digest extracted from RepoDigests
  </behavior>
  <implementation>
**action/Dockerfile** — Magic Dockerfile per RESEARCH.md Section 3:
- `# syntax=docker/dockerfile:1`
- `ARG PYTHON_VERSION=3.12`
- `FROM public.ecr.aws/lambda/python:${PYTHON_VERSION}`
- COPY glob trick for optional system-requirements.txt: `COPY system-requirements.tx[t] /tmp/`
- Conditional RUN for system packages: `if [ -f /tmp/system-requirements.txt ]; then dnf install -y $(cat /tmp/system-requirements.txt) && dnf clean all; fi`
- COPY glob trick for optional system-config.sh: `COPY system-config.s[h] /tmp/`
- Conditional RUN for system config: `if [ -f /tmp/system-config.sh ]; then chmod +x /tmp/system-config.sh && /tmp/system-config.sh; fi`
- `COPY requirements.txt ${LAMBDA_TASK_ROOT}/`
- RUN with secret mount for pip install: `--mount=type=secret,id=github_token` with conditional git config for private repos, then `pip install --no-cache-dir -r ${LAMBDA_TASK_ROOT}/requirements.txt`
- `COPY . ${LAMBDA_TASK_ROOT}/` — copies entire source_dir
- `CMD ["main.handler"]`

**action/src/ferry_action/build.py** — Build module:
- Import subprocess, os, and ferry_action.gha
- get_ecr_uri: simple f-string construction
- ecr_login: subprocess.run for AWS ECR get-login-password piped to docker login. Use the registry portion of the ECR URI (without the repo path). Mask the login password.
- build_image: construct docker build command list, run with subprocess.run(check=True, capture_output=True, text=True). Print stdout inside gha.group(). On CalledProcessError, check failure_hint and re-raise.
- push_image: subprocess.run for docker push, then docker inspect for digest extraction. Parse "repo@sha256:..." format, return just "sha256:..." portion.

**Tests — mock subprocess.run, do NOT run actual Docker commands:**
- test_get_ecr_uri_format: verify URI construction
- test_build_image_command_basic: mock subprocess, verify command args include correct --build-arg, --tag, --file, source_dir
- test_build_image_command_with_secret: verify --secret flag added when github_token provided
- test_build_image_command_no_secret_when_empty: verify no --secret when github_token=""
- test_build_image_runtime_strip: "python3.14" -> build-arg has "3.14"
- test_ecr_login_command: verify aws ecr get-login-password piped to docker login with correct registry
- test_push_image_returns_digest: mock subprocess with RepoDigest output, verify digest extraction
- test_push_image_command: verify docker push called with correct image tag
- test_build_image_failure_with_hint: mock subprocess to raise CalledProcessError with pip-related stderr, verify failure_hint is called
