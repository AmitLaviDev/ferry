**RED phase — Write tests first (`tests/test_action/test_build.py`):**

Test the build module by mocking subprocess calls (never actually invoke docker in tests). Extract the core logic into testable functions:

- `build_ecr_uri(aws_account_id: str, aws_region: str, ecr_repo: str) -> str` — constructs `{account}.dkr.ecr.{region}.amazonaws.com/{repo}`
- `build_docker_command(dockerfile_path: str, source_dir: str, image_tag: str, python_version: str, github_token: str | None) -> list[str]` — returns the full docker build command as a list
- `parse_runtime_version(runtime: str) -> str` — extracts version number from "python3.12" -> "3.12"
- `ecr_login(aws_region: str)` — constructs ECR login command
- `push_image(image_tag: str) -> str` — pushes and returns digest
- `main()` — orchestrator that reads env vars and calls the above

**GREEN phase — Implement:**

1. **`action/Dockerfile`** — The Magic Dockerfile (from RESEARCH.md, verbatim):
```dockerfile
# syntax=docker/dockerfile:1
ARG PYTHON_VERSION=3.12
FROM public.ecr.aws/lambda/python:${PYTHON_VERSION}

# Optional system packages
COPY system-requirements.tx[t] /tmp/
RUN if [ -f /tmp/system-requirements.txt ]; then \
      dnf install -y $(cat /tmp/system-requirements.txt) && dnf clean all; \
    fi

# Optional system config script
COPY system-config.s[h] /tmp/
RUN if [ -f /tmp/system-config.sh ]; then \
      chmod +x /tmp/system-config.sh && /tmp/system-config.sh; \
    fi

# Install Python dependencies
COPY requirements.txt ${LAMBDA_TASK_ROOT}/
RUN --mount=type=secret,id=github_token \
    if [ -f /run/secrets/github_token ]; then \
      export GIT_TOKEN=$(cat /run/secrets/github_token) && \
      git config --global url."https://${GIT_TOKEN}@github.com/".insteadOf "https://github.com/" ; \
    fi && \
    pip install --no-cache-dir -r ${LAMBDA_TASK_ROOT}/requirements.txt

# Copy entire source directory
COPY . ${LAMBDA_TASK_ROOT}/

CMD ["main.handler"]
```

2. **`action/src/ferry_action/build.py`** — Build module:

   - `parse_runtime_version(runtime: str) -> str`: Strip "python" prefix if present.
   - `build_ecr_uri(aws_account_id: str, aws_region: str, ecr_repo: str) -> str`: Construct full ECR URI.
   - `build_docker_command(...)`: Build the `docker build` command list with:
     - `--build-arg PYTHON_VERSION={version}`
     - `--secret id=github_token,env=GITHUB_TOKEN` (only if github_token provided)
     - `--tag {ecr_uri}:{deployment_tag}`
     - `--file {dockerfile_path}` (use `Path(__file__).resolve().parent.parent.parent / "Dockerfile"` to locate the Dockerfile relative to the module — action/Dockerfile is two levels up from action/src/ferry_action/)
     - Build context = source_dir (last arg)
   - `ecr_login(aws_region: str, ecr_uri: str)`: Run `aws ecr get-login-password --region {region}` and pipe to `docker login --username AWS --password-stdin {registry}` where registry is the ECR domain (everything before the repo name).
   - `push_image(image_tag: str) -> str`: Run `docker push {image_tag}`, then `docker inspect --format='{{index .RepoDigests 0}}' {image_tag}` to get the digest.
   - `main()`:
     a. Read env vars: INPUT_RESOURCE_NAME, INPUT_SOURCE_DIR, INPUT_ECR_REPO, INPUT_DEPLOYMENT_TAG, INPUT_RUNTIME, INPUT_TRIGGER_SHA, INPUT_GITHUB_TOKEN
     b. Get AWS account ID via `boto3.client("sts").get_caller_identity()["Account"]`
     c. Get AWS region from `AWS_REGION` or `AWS_DEFAULT_REGION` env var (set by configure-aws-credentials)
     d. Call `gha.mask_value(account_id)` to mask AWS account ID in logs
     e. Build ECR URI
     f. `gha.begin_group(f"Building {resource_name}")`
     g. Run docker build command via `subprocess.run(cmd, check=True)`
     h. `gha.end_group()`
     i. `gha.begin_group(f"Pushing {resource_name} to ECR")`
     j. ECR login
     k. Push image, capture digest
     l. `gha.end_group()`
     m. Set outputs: `gha.set_output("image-uri", f"{ecr_uri}:{deployment_tag}")` and `gha.set_output("image-digest", digest)`
     n. Write job summary via `gha.write_summary(...)` with resource name, ECR tag, trigger SHA
   - Use `subprocess.run` with `check=True` for all shell commands. Wrap in try/except to provide common failure hints (per CONTEXT.md decisions):
     - `FileNotFoundError` for missing docker → hint about docker not being installed
     - `CalledProcessError` with stderr pattern matching for known failures
   - Run as `__main__`: support `python -m ferry_action.build`
