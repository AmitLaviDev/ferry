  <behavior>
    Deploy module handles the Lambda update-function-code -> wait -> publish-version -> update-alias sequence:

    deploy_lambda(function_name, image_uri, alias_name="live") -> dict with {"version": str, "alias": str}
    - Calls lambda_client.update_function_code(FunctionName=function_name, ImageUri=image_uri)
    - Waits using lambda_client.get_waiter("function_updated").wait(FunctionName=function_name)
    - Publishes version: lambda_client.publish_version(FunctionName=function_name, Description=f"Deployed by Ferry: {image_uri}")
    - Updates alias: lambda_client.update_alias(FunctionName=function_name, Name=alias_name, FunctionVersion=new_version)
      If alias doesn't exist (ResourceNotFoundException), creates it: lambda_client.create_alias(...)
    - Returns {"version": new_version, "alias": alias_name}

    get_current_image_digest(function_name) -> str | None
    - Calls lambda_client.get_function(FunctionName=function_name)
    - Reads Code.ResolvedImageUri (format: "repo@sha256:...")
    - Extracts and returns "sha256:..." portion
    - Returns None if function has no image-based code (catches KeyError)

    Digest module:

    should_skip_deploy(function_name, new_digest) -> bool
    - Calls get_current_image_digest(function_name)
    - If current_digest is None, returns False (no existing image, must deploy)
    - If current_digest == new_digest, returns True (skip)
    - Otherwise returns False (image changed, must deploy)

    Main orchestrator (action/src/ferry_action/main.py):

    run(resource_name, source_dir, ecr_repo, aws_role_arn, deployment_tag, trigger_sha, runtime, github_token="") -> None
    - Reads AWS account ID from STS get-caller-identity
    - Masks AWS account ID and role ARN via gha.mask_value()
    - Constructs ECR URI via build.get_ecr_uri()
    - Logs: "Building {resource_name} (runtime: {runtime})"
    - Calls build.ecr_login(ecr_uri)
    - Calls build.build_image() -> image_tag
    - Calls build.push_image(image_tag) -> digest
    - Calls digest.should_skip_deploy(resource_name, digest)
    - If skip: log "Skipping deploy for {resource_name} — image unchanged", set output skipped=true, write summary with skipped=True
    - If not skip: calls deploy.deploy_lambda(resource_name, f"{ecr_uri}:{deployment_tag}") -> result
    - Writes job summary via gha.write_summary(gha.format_resource_summary(...))
    - On any exception: log error, call gha.failure_hint(), write error summary, exit(1)

    if __name__ == "__main__" block:
    - Reads INPUT_RESOURCE_NAME, INPUT_SOURCE_DIR, INPUT_ECR_REPO, INPUT_DEPLOYMENT_TAG, INPUT_TRIGGER_SHA, INPUT_RUNTIME, INPUT_GITHUB_TOKEN from env
    - Calls run() with these values
    - Note: aws_role_arn is not needed here because OIDC auth happens in the composite action step before this script runs

    Cases:
    - deploy_lambda("order-processor", "123.dkr.ecr.us-east-1.amazonaws.com/ferry/order:pr-42") -> updates code, waits, publishes version "1", creates/updates "live" alias
    - get_current_image_digest("order-processor") with image-based function -> "sha256:abc..."
    - get_current_image_digest("order-processor") with non-image function -> None
    - should_skip_deploy("order-processor", "sha256:abc...") when current is same -> True
    - should_skip_deploy("order-processor", "sha256:xyz...") when current differs -> False
  </behavior>

**action/src/ferry_action/deploy.py**:
- Import boto3, ferry_action.gha
- Create lambda_client = boto3.client("lambda") at module level
- deploy_lambda: update_function_code -> get_waiter("function_updated").wait -> publish_version -> try update_alias, except ClientError (alias not found) create_alias. Wrap in gha.group("Deploying {function_name}"). Return version and alias info.
- get_current_image_digest: get_function -> Code.ResolvedImageUri -> split on "@" -> return digest part. Return None on KeyError.

**action/src/ferry_action/digest.py**:
- Import deploy.get_current_image_digest
- should_skip_deploy: compare current vs new digest. Log comparison result.

**action/src/ferry_action/main.py**:
- Import os, sys, time, boto3
- Import from ferry_action: build, deploy, digest, gha
- run() function with full orchestration flow as described in behavior
- __main__ block reads env vars and calls run()
- Wrap entire run in try/except, failure path logs error and hint
