**RED phase — Write tests first (`tests/test_action/test_deploy.py`):**

Use `moto`'s `@mock_aws` decorator for Lambda and ECR operations. Extract deploy logic into testable functions:

- `get_current_image_digest(lambda_client, function_name: str) -> str | None` — gets the current Lambda's resolved image digest
- `should_skip_deploy(current_digest: str | None, new_digest: str) -> bool` — digest comparison
- `deploy_lambda(lambda_client, function_name: str, image_uri: str, deployment_tag: str, alias_name: str) -> dict` — full deploy sequence returning result dict
- `main()` — orchestrator reading env vars

**Test fixtures:** Create a helper that sets up a moto Lambda function with a container image. Note: moto may not fully support `ImageUri` in Lambda — if so, test the logic flow with what moto supports and mock `get_function` for digest checks where needed.

2. **test_should_skip_deploy_different_digest**: `should_skip_deploy("sha256:abc123", "sha256:def456")` returns `False`.

3. **test_should_skip_deploy_no_current**: `should_skip_deploy(None, "sha256:abc123")` returns `False` (first deploy, no current image).

4. **test_deploy_lambda_updates_function_code**: With moto Lambda, create a function, call `deploy_lambda(...)`, verify `get_function` shows updated image URI.

5. **test_deploy_lambda_publishes_version**: After deploy, verify a new version was published via `list_versions_by_function`. The version description should contain the deployment tag.

6. **test_deploy_lambda_updates_alias**: After deploy, verify the `live` alias exists and points to the new version. Use `get_alias(FunctionName=..., Name="live")`.

7. **test_deploy_lambda_creates_alias_if_not_exists**: On first deploy (no alias yet), `deploy_lambda` should create the alias via `create_alias` (not `update_alias`). Catch `ResourceNotFoundException` and fall back to `create_alias`.

8. **test_deploy_lambda_returns_result**: Verify `deploy_lambda` returns a dict with keys: `version` (str), `alias` (str), `skipped` (bool=False).

9. **test_main_skips_when_digest_matches**: Set env vars, mock `get_function` to return matching digest, mock GITHUB_OUTPUT file. Verify main() writes `skipped=true` output and does NOT call `update_function_code`.

10. **test_main_deploys_when_digest_differs**: Set env vars with a new digest, mock Lambda client. Verify main() calls the full deploy sequence and writes `skipped=false` and `lambda-version={N}` outputs.

11. **test_main_writes_job_summary**: Verify main() writes markdown to GITHUB_STEP_SUMMARY with resource name, Lambda version, deploy status, and deployment tag.

12. **test_main_masks_nothing_extra**: Deploy action doesn't need to mask account ID (build action already did). Verify no unnecessary masking.

Use `monkeypatch` for env vars, `@mock_aws` for boto3, `tmp_path` for GITHUB_OUTPUT and GITHUB_STEP_SUMMARY.

**GREEN phase — Implement `action/src/ferry_action/deploy.py`:**

1. **`get_current_image_digest(lambda_client, function_name: str) -> str | None`**:
   - Call `lambda_client.get_function(FunctionName=function_name)`
   - Extract `Code.ResolvedImageUri` — contains `@sha256:...`
   - Parse out the digest part (everything after `@`)
   - Return `None` if function has no image config or key is missing
   - Wrap in try/except for `ClientError` (function might not exist yet — return None)

2. **`should_skip_deploy(current_digest: str | None, new_digest: str) -> bool`**:
   - Normalize both digests: extract just the `sha256:...` part if full URI is given
   - Return `current_digest is not None and current_digest == new_digest`

3. **`deploy_lambda(lambda_client, function_name: str, image_uri: str, deployment_tag: str, alias_name: str = "live") -> dict`**:
   - Step 1: `update_function_code(FunctionName=function_name, ImageUri=image_uri)`
   - Step 2: Wait for function update: `lambda_client.get_waiter("function_updated").wait(FunctionName=function_name)` — wrap in try/except WaiterError for timeout with clear error message
   - Step 3: `publish_version(FunctionName=function_name, Description=f"Deployed by Ferry: {deployment_tag}")` — capture version number from response
   - Step 4: Try `update_alias(FunctionName=function_name, Name=alias_name, FunctionVersion=version)`. On `ResourceNotFoundException`, fall back to `create_alias(FunctionName=function_name, Name=alias_name, FunctionVersion=version)`.
   - Return `{"version": version, "alias": alias_name, "skipped": False}`

4. **`main()`**:
   - Read env vars: INPUT_RESOURCE_NAME, INPUT_FUNCTION_NAME, INPUT_IMAGE_URI, INPUT_IMAGE_DIGEST, INPUT_DEPLOYMENT_TAG
   - Create `boto3.client("lambda")`
   - `gha.begin_group(f"Deploying {resource_name}")`
   - Get current digest: `get_current_image_digest(client, function_name)`
   - Check skip: `should_skip_deploy(current_digest, new_digest)`
   - If skip:
     - Log: `print(f"Skipping deploy for {resource_name} -- image unchanged")`
     - `gha.set_output("skipped", "true")`
     - `gha.set_output("lambda-version", "")` (no new version)
     - Write summary indicating skipped
     - `gha.end_group()`
     - Return
   - If not skip:
     - Call `deploy_lambda(client, function_name, image_uri, deployment_tag)`
     - `gha.set_output("skipped", "false")`
     - `gha.set_output("lambda-version", result["version"])`
     - Write per-resource job summary:
       ```markdown
       ## {resource_name}
       | Field | Value |
       |---|---|
       | Function | `{function_name}` |
       | ECR Image | `{image_uri}` |
       | Lambda Version | `{version}` |
       | Alias | `live` |
       | Status | Deployed |
       | Deployment Tag | `{deployment_tag}` |
       ```
     - `gha.end_group()`
   - Error handling: catch `ClientError` from boto3, provide hints for common failures:
     - AccessDeniedException → "IAM role lacks lambda:UpdateFunctionCode permission"
     - ResourceNotFoundException → "Lambda function '{function_name}' not found"
     - WaiterError → "Lambda update timed out after 5 minutes"
   - Run as `__main__`: support `python -m ferry_action.deploy`
