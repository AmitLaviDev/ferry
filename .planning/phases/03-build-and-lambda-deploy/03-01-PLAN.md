Create GHA-specific logging and output helpers that Plans 02 and 03 will use:

**action/src/ferry_action/gha.py**:
- `group(title: str)` — context manager that prints `::group::{title}` on enter and `::endgroup::` on exit. Use `contextlib.contextmanager`.
- `mask_value(value: str)` — prints `::add-mask::{value}` to mask sensitive values in logs.
- `set_output(name: str, value: str)` — appends `{name}={value}` to `GITHUB_OUTPUT` file (read path from env var, with fallback to stdout for local testing).
- `write_summary(markdown: str)` — appends markdown to `GITHUB_STEP_SUMMARY` file (read path from env var, with fallback to stdout for local testing).
- `failure_hint(error_msg: str) -> str | None` — pattern-match common failure messages and return a one-liner remediation hint. Patterns:
  - "No such file" + "main.py" -> "Lambda handler not found. Ensure main.py exists in source_dir with a handler() function."
  - "No matching distribution" or "Could not find" in pip context -> "pip install failed. Check requirements.txt for syntax errors or unavailable packages."
  - "authorization" or "401" in ECR context -> "ECR login failed. Verify the IAM role has ecr:GetAuthorizationToken permission."
  - "ResourceNotFoundException" for Lambda -> "Lambda function not found. Verify the function exists and the name matches ferry.yaml."
  - Return None for unrecognized patterns (raw error is sufficient).
- `format_resource_summary(resource_name: str, ecr_tag: str, lambda_version: str | None, status: str, duration_s: float, skipped: bool = False) -> str` — returns markdown table for GHA job summary per CONTEXT.md spec:
  ```
  ## {resource_name}
  | Field | Value |
  |---|---|
  | ECR Tag | `{ecr_tag}` |
  | Lambda Version | `{lambda_version}` |
  | Status | {status} |
  | Duration | {duration_s}s |
  ```
  If skipped=True, status should be "Skipped (image unchanged)".
