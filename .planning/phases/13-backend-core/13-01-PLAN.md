**providers.tf:**
- S3 backend: bucket `ferry-global-terraform-state`, key `aws/staging/us-east-1/ferry_backend/terraform.tfstate`, region `us-east-1`, `use_lockfile = true`
- Required version: `~> 1.12.0`
- AWS provider `~> 6.0` with `var.region` and `default_tags` (ManagedBy=terraform, Project=ferry)
- Include the same "No assume_role" comment as shared/providers.tf

**data.tf:**
- `terraform_remote_state.shared`: bucket `ferry-global-terraform-state`, key `aws/staging/shared/terraform.tfstate`, region `us-east-1` (this is the NEW key from Phase 12.1)
- `terraform_remote_state.ecr`: bucket `ferry-global-terraform-state`, key `global/cloud/aws/ecr/terraform.tfstate`, region `us-east-1`
- No `aws_caller_identity` needed — all account-specific values come from remote state

**main.tf** (all 4 resources in one file — they are tightly coupled, splitting is over-engineering for 4 resources):

1. `aws_cloudwatch_log_group.backend`: name `/aws/lambda/ferry-backend`, `retention_in_days = 30`, tag Name = `ferry-backend-logs`

2. `aws_dynamodb_table.dedup`: name `ferry-webhook-dedup`, `billing_mode = "PAY_PER_REQUEST"`, `hash_key = "pk"`, `range_key = "sk"`, two `attribute` blocks for pk (S) and sk (S) ONLY (do NOT define expires_at as an attribute — it is schemaless, only referenced in the ttl block). TTL block: `attribute_name = "expires_at"`, `enabled = true`. Tag Name = `ferry-webhook-dedup`.

3. `aws_lambda_function.backend`: function_name `ferry-backend`, role from `data.terraform_remote_state.shared.outputs.lambda_execution_role_arn`, `package_type = "Image"`, `image_uri = "${data.terraform_remote_state.ecr.outputs.repository_url}:latest"`, `architectures = ["arm64"]`, `memory_size = 256`, `timeout = 30`. Environment variables block:
   - `FERRY_APP_ID_SECRET = "ferry/github-app/app-id"` (hardcoded secret name per RESEARCH.md discretion decision — deterministic from secrets.tf for_each keys)
   - `FERRY_PRIVATE_KEY_SECRET = "ferry/github-app/private-key"`
   - `FERRY_WEBHOOK_SECRET_SECRET = "ferry/github-app/webhook-secret"`
   - `FERRY_TABLE_NAME = aws_dynamodb_table.dedup.name` (from TF resource, NOT hardcoded)
   - `FERRY_LOG_LEVEL = var.log_level`
   - `FERRY_INSTALLATION_ID = var.installation_id` (placeholder TF variable, per RESEARCH.md discretion decision)
   Add `depends_on = [aws_cloudwatch_log_group.backend]` to ensure log group exists before Lambda.
   Add `lifecycle { ignore_changes = [image_uri] }` so Terraform owns infra, GHA owns deployed code.
   Tag Name = `ferry-backend`.

4. `aws_lambda_function_url.backend`: `function_name = aws_lambda_function.backend.function_name`, `authorization_type = "NONE"`.
