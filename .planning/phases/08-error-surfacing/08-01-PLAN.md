def patch(self, path: str, **kwargs: Any) -> httpx.Response:

1. **handler.py** -- Restructure the handler to add a top-level exception handler and switch config errors from Check Run to PR comment.

   a. Add import for `GitHubAuthError` from `ferry_utils.errors` and import `find_merged_pr, post_pr_comment` from `checks.runs` (alongside existing imports).

   b. Wrap the entire handler body (from step 8 onward -- after dedup) in a `try` block with three catch levels:
      - `except ConfigError as exc:` -- Already exists but change behavior:
        - Log with `log.error("config_error", error=str(exc))`
        - **PR branches**: Call `find_open_prs`, if PRs found, call `post_pr_comment` with a formatted markdown body: `"**Ferry: Configuration Error**\n\nferry.yaml validation failed:\n```\n{str(exc)}\n```"`
        - **Default branch**: Call `find_merged_pr`, if a merged PR found, call `post_pr_comment` with same body. If no PR found, log a warning -- the error is in backend logs only.
        - Do NOT create a Check Run for config errors (per user decision). Remove the existing `create_check_run(..., error=str(exc))` call.
        - Return `_response(200, {"status": "config_error", "error": str(exc)})`
      - `except GitHubAuthError as exc:` -- NEW:
        - Log with `log.error("auth_error", error=str(exc), exc_info=True)`
        - Return `_response(500, {"status": "auth_error", "error": str(exc)})`
        - No PR comment, no Check Run (per user decision -- auth errors are infra-visible only)
      - `except Exception as exc:` -- NEW (catch-all):
        - Log with `log.error("unhandled_error", error=str(exc), exc_info=True)`
        - Return `_response(500, {"status": "internal_error", "error": "internal server error"})` (do NOT leak exception details in response)

   c. The ConfigError handler placement: Move the try/except to wrap steps 8-13 (after dedup, where the real work begins). Steps 1-7 (body extraction, sig validation, dedup) can remain outside since they have their own explicit error handling already.

   d. Remove the `error` parameter from `create_check_run` in its import and all calls in handler.py since config errors no longer use Check Runs. The `create_check_run` function itself keeps the `error` parameter for backward compatibility but handler.py no longer passes it.

   - Update any existing test that verifies config error creates a Check Run -- change assertion to verify PR comment instead
