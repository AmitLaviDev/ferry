# Phase 12.1: IaC Directory Restructure and State Migration - Context

**Gathered:** 2026-03-02
**Status:** Ready for planning

<domain>
## Phase Boundary

Reorganize existing Terraform directory layout to match ConvergeBio/iac-tf conventions and migrate all remote state to new S3 keys. No new resources, no project splits — pure directory restructuring and state migration.

</domain>

<decisions>
## Implementation Decisions

### Target directory layout
- 4 directory moves, no project splits
- Global: `iac/global/aws/{backend,ecr,oidc}` → `iac/global/cloud/aws/{backend,ecr,oidc}`
- Staging: `iac/staging/aws/shared/` → `iac/aws/staging/shared/` (single project, NOT split)
- `iac/resources/placeholders/` stays in place (not a TF project)
- Cloud provider level added for global: `global/cloud/aws/`
- Cloud provider first for staging: `aws/staging/` instead of `staging/aws/`

### Shared project — no split
- Shared project keeps all files together: iam.tf, secrets.tf, data.tf, outputs.tf, variables.tf, providers.tf
- No separation into iam/ and secrets/ sub-projects
- Phase 13 will reference shared's remote state as a single source

### S3 state key convention
- Keys mirror directory paths exactly
- `global/cloud/aws/backend/terraform.tfstate`
- `global/cloud/aws/ecr/terraform.tfstate`
- `global/cloud/aws/oidc/terraform.tfstate`
- `aws/staging/shared/terraform.tfstate`

### State migration approach
- `terraform init -migrate-state` for each project (built-in TF migration)
- Update backend key in providers.tf, then run init
- Update all `terraform_remote_state` references to use new S3 keys (e.g., shared's OIDC remote state ref)
- Migration script (shell script) handles all 4 moves sequentially
- Delete old `.terraform/` directories before reinit — clean slate

### Apply order and validation
- Globals migrate first (backend → ecr → oidc), then shared
- OIDC must complete before shared (shared references OIDC remote state)
- Validation: `terraform plan` per project after migration — must show "No changes"
- Fail fast on first failure, manual rollback (no automatic revert)

### Claude's Discretion
- Migration script structure and error messaging
- Whether to back up state files before migration
- .terraform.lock.hcl handling during moves

</decisions>
