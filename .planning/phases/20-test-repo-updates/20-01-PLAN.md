**workflows/hello-chain/definition.asl.json** -- Minimal ASL definition with a single Task state that invokes the test Lambda:

```json
{
  "StartAt": "InvokeLambda",
  "States": {
    "InvokeLambda": {
      "Type": "Task",
      "Resource": "arn:aws:lambda:${AWS_REGION}:${ACCOUNT_ID}:function:ferry-test-hello-world",
      "End": true
    }
  }
}
```

Key design choices (from CONTEXT.md):
- Single Task state → invoke Lambda → End (minimal surface area)
- RequestResponse invocation (default for Task states -- SF waits for Lambda result)
- Passthrough input handling (whatever SF receives is forwarded to Lambda)
- No Catch/Retry states (minimize debug surface for E2E validation)
- `${ACCOUNT_ID}` and `${AWS_REGION}` placeholders substituted at deploy time by `envsubst.py`

**api/hello-chain/openapi.yaml** -- Swagger 2.0 spec with POST /execute endpoint that triggers the Step Function:

```yaml
swagger: "2.0"
info:
  title: "Ferry Test API"
  version: "1.0"
paths:
  /execute:
    post:
      summary: "Start Step Function execution"
      consumes:
        - application/json
      produces:
        - application/json
      responses:
        "200":
          description: "Execution started"
      x-amazon-apigateway-integration:
        type: aws
        httpMethod: POST
        uri: "arn:aws:apigateway:${AWS_REGION}:states:action/StartExecution"
        credentials: "arn:aws:iam::${ACCOUNT_ID}:role/ferry-test-apgw-invoke"
        requestTemplates:
          application/json: |
            {
              "stateMachineArn": "arn:aws:states:${AWS_REGION}:${ACCOUNT_ID}:stateMachine:ferry-test-sf",
              "input": "$util.escapeJavaScript($input.body)"
            }
        responses:
          default:
            statusCode: "200"
```

Key design choices (from CONTEXT.md):
- Swagger 2.0 (best AWS API Gateway REST API native support)
- Integration type `aws` (not `aws_proxy`) -- uses request/response templates for explicit input mapping
- URI: `arn:aws:apigateway:{region}:states:action/StartExecution` -- standard service integration URI
- Credentials: APGW invoke role ARN (role created in Phase 19, has `states:StartExecution` permission)
- Request template: maps HTTP body to SF `input` via `$util.escapeJavaScript()` (VTL function, safe from envsubst)
- Response: passthrough (raw StartExecution response with executionArn, startDate)
- No `host`, `schemes`, `basePath` fields (these are stripped by deploy_apigw.py anyway)
- `${ACCOUNT_ID}` and `${AWS_REGION}` in URI, credentials, and request template are all substituted by envsubst

**ferry-step_functions.yml** -- Adapted from `docs/step-functions.md` template with external composite action references:

```yaml
# .github/workflows/ferry-step_functions.yml
# Ferry Step Functions deployment workflow
# Triggered by Ferry App via workflow_dispatch when definition files change

name: Ferry Step Functions

on:
  workflow_dispatch:
    inputs:
      payload:
        description: "Ferry dispatch payload (JSON)"
        required: true

permissions:
  id-token: write
  contents: read
  checks: write

jobs:
  setup:
    runs-on: ubuntu-latest
    outputs:
      matrix: ${{ steps.parse.outputs.matrix }}
    steps:
      - uses: actions/checkout@v4

      - name: Parse Ferry payload
        id: parse
        uses: AmitLaviDev/ferry/action/setup@main
        with:
          payload: ${{ inputs.payload }}

  deploy:
    needs: setup
    runs-on: ubuntu-latest
    strategy:
      matrix: ${{ fromJson(needs.setup.outputs.matrix) }}
      fail-fast: false
    steps:
      - uses: actions/checkout@v4

      - name: Deploy Step Functions
        uses: AmitLaviDev/ferry/action/deploy-stepfunctions@main
        with:
          resource-name: ${{ matrix.name }}
          state-machine-name: ${{ matrix.state_machine_name }}
          definition-file: ${{ matrix.definition_file }}
          source-dir: ${{ matrix.source }}
          trigger-sha: ${{ matrix.trigger_sha }}
          deployment-tag: ${{ matrix.deployment_tag }}
          aws-role-arn: ${{ secrets.AWS_ROLE_ARN }}
          aws-region: us-east-1
          github-token: ${{ github.token }}
```
