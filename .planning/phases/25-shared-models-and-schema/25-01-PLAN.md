    Add the `BatchedDispatchPayload` class to `dispatch.py` after the existing `DispatchPayload` class. Import the new constant.

    **Model definition:**
    ```python
    from ferry_utils.constants import BATCHED_SCHEMA_VERSION, SCHEMA_VERSION

    class BatchedDispatchPayload(BaseModel):
        """Batched payload sent via workflow_dispatch from Ferry App to Ferry Action.

        v2 schema: One payload per push containing ALL affected resource types,
        grouped into typed lists. Replaces per-type DispatchPayload (v1) to
        eliminate multiple workflow runs per push.
        """

        model_config = ConfigDict(frozen=True)

        v: Literal[2] = BATCHED_SCHEMA_VERSION
        lambdas: list[LambdaResource] = []
        step_functions: list[StepFunctionResource] = []
        api_gateways: list[ApiGatewayResource] = []
        trigger_sha: str
        deployment_tag: str
        pr_number: str = ""
    ```

    Key design decisions:
    - `v: Literal[2]` (not `int`) — enforces that v2 payloads always have v=2, enabling reliable version detection via Pydantic discriminated unions in the action parser (phase 27)
    - Typed lists per resource type (not a generic `resources` list) — the action can directly iterate each type without filtering
    - All three type lists default to empty — a batched payload may contain any subset of types
    - Same common fields as v1 (trigger_sha, deployment_tag, pr_number) for consistency
    - Frozen model for immutability, matching v1 convention

    Do NOT modify the existing `DispatchPayload` class or any existing code above it.
  </action>
